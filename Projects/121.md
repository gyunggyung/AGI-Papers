---

며칠 전 H200 8대로 수십 개의 터미널 에이전트 모델을 학습하고 벤치마크 점수만 믿으면 안 된다는 글을 남겼습니다. 이후 SFT 파이프라인과 평가 코드의 오류들을 바로잡고, ByteDance의 재귀 언어 모델인 Ouro 모델까지 새롭게 추가하여 56개 모델 전면 재평가를 진행했습니다.

단순히 순위를 매기는 것을 넘어, 파라미터 수보다 강력한 아키텍처의 특성과 데이터 포맷이 빚어낸 이야기를 공유합니다.

1. 생태계 교란종 Qwen, 그리고 LFM

새로운 평가에서 Qwen3.5-2B가 39.52점으로 35B 대형 모델들을 다 꺾고 전체 1위를 차지했습니다. Qwen 3.5는 베이스 모델 자체가 터미널 명령어와 JSON 포맷 출력에 매우 강합니다. 

SFT 학습은 그저 거들 뿐, 베이스 모델의 성능 자체가 타 모델들이 SFT를 끝마친 수준입니다.

반면 LFM(Liquid AI)은 효율적이고 매력적인 모델입니다. 베이스 점수는 낮았지만 SFT 학습 후 무려 +15~22점이 폭발적으로 올랐습니다. Qwen 3.5에 비해서 학습은 덜 됐지만, SFT 과정으로 충분히 좋아지는 모습을 보입니다. 

비록 단일 스텝 추론 속도는 Qwen에 살짝 밀리지만, VRAM을 적게 먹고 모델 로딩 속도가 압도적으로 빨라 효율이 좋습니다.

2. 억울했던 Gemma 4 (범인은 전처리였다)

저번 실험에서 대형 Gemma 4(26B, 31B) 모델들이 학습 후 오히려 바보가 됐다고 했었죠. 지능 문제가 아니었습니다. 

과거 대화의 <think> 블록과 Empty thought channel 같은 템플릿 요소를 제대로 정리하지 않고 학습시킨 포맷 충돌이 원인이었습니다.

네이티브 템플릿으로 전처리를 수정하고 다시 평가를 돌려보니 점수가 정상적으로 크게 뛰어올랐습니다. 그리고 학습 자체도 문제가 있어서 다시 학습을 돌리고 있습니다. 

결국 모델 아키텍처보다 데이터 퀄리티와 정확한 포맷팅이 결과를 지배합니다.

3. 똑똑하지만 너무 느린 천재, Ouro

Ouro 2.6B-Thinking 모델은 SFT 이후 35.61점을 기록하며 LFM의 최고점(33.46)을 넘는 대단한 지능을 보여줬습니다. 하지만 터미널 환경에선 쥐약입니다. 

24층을 4번 반복 연산하는 구조 탓에 같은 체급의 LFM 대비 무려 34배나 느립니다. 게다가 포맷 안정성이 떨어져 억지로 JSON을 뱉어내는 수준입니다.

따라서 실시간 피드백 루프가 생명인 강화학습 단계에서는 제외했습니다.


앞으로의 계획: 강화학습의 시간

모델들에게 "JSON으로 이렇게 대답해"라고 떠먹여 주는 SFT 작업은 끝났습니다. 프록시 평가 역시 짧고 정확한 단기 명령에만 유리한 편향이 있어, 큰 모델들의 진짜 잠재력(에러 복구, 장기 계획)을 측정하지 못하고 있습니다.

SFT 파이프라인에서 살아남은 소수 정예(최고점의 Qwen, 효율적인 LFM)를 실제 터미널 환경에 던져놓을 차례입니다. 이제 강화학습(RLVR) 만으로 에러를 직접 맞아가며 스스로 복구하는 방법을 터득하게 만들 계획입니다.
