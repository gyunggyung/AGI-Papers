---

어제 Hugging Face 다운로드 수가 1,000회를 넘어서 기쁘다고 썼고, 오늘 제가 학습한 터미널 모델들의 다운로드 수 총합이 27,868회입니다. 

3개월 전에는 허깅페이스에 내 이름의 공간을 만들고, 모델을 올린다는 것 자체도 상상하지 못했던 초보였습니다. 

자원이 부족하고 인프라가 없는 이른바 'GPU Poor' 환경에서, 제가 좋아하는 이 인공지능 공부를 포기하지 않고 가치를 증명할 수 있었던 3가지 작은 소회를 적어봅니다.

1. "현실을 직시하고 틈새를 찔렀습니다."

H100 수천 대를 가진 대기업과 프론티어 랩들이 100B, 400B 크기의 거대 모델들을 쏟아내며 힘자랑을 하고 있을 때, 제가 그곳에 비빌 리소스는 단 1%도 없었습니다. 

그래서 저는 '온디바이스(On-device)'와 '소형 특화 모델(sLM)'로 시선을 돌렸습니다. 1B, 3B 크기의 모델이 내 노트북 CPU만으로 한국어를 리눅스 명령어로 바꿔주는Yaongi/HybriKo 모델, 그리고 350M 초소형 모델로 API 호출 없이 로컬에서 즉시 작동하는 도구(PicoClaw)를 만들었습니다. 

거인들이 쳐다보지 않는 작고 더러운 틈새에 저의 온 힘을 쏟았습니다.

2. "오픈소스 생태계에 먼저 아낌없이 뿌렸습니다."

기술은 혼자 쥐고 있으면 썩어버립니다. 
저는 HybriKo 아키텍처 설계PoC 노트북부터, 데이터 전처리 스크립트, 그리고 번역기 웹 UI까지 개발하는 모든 과정을 깃허브(AGI-Papers)에 오픈소스로 투명하게 공개했습니다. 

그랬더니 신기하게도 똑똑한 분들이 피드백을 주며 찾아와 주셨고, 김정수님 같은 든든한 조력자분께서 H200 GPU 서버를 지원해 주시는 기적으로 이어졌습니다. 나누는 것이 결국 가장 빠르게 배우고 얻는 길입니다.

3. "벤치마크 숫자보다 실무 검증에 목숨을 걸었습니다."

점수판에서 소수점 몇 점 더 올리는 튜닝 장난질은 의미가 없습니다. 
저는 모델이 완성되면 항상 한국어로 리눅스 명령어를 직접 실행해 보고, Streamlit으로 웹 페이지를 만들어 돌려보며 '실제 쓸만한가?'를 끊임없이 스스로 괴롭히며 검증했습니다. 이 집요함이 다운로드 수라는 실제 사용량으로 증명되었다고 생각합니다.


마치며: 가난은 핑계일 뿐입니다.

리소스가 없어서, 자본이 없어서 연구를 못 한다는 것은 핑계일 뿐입니다. 

내가 가진 작은 맥북 한 대, 혹은 무료 코랩 서버 한 장이라도 그 한계치까지 뜯어보고 최적화하겠다는 독기만 있다면, 거대한 빅테크들이 놓치는 가치 있는 영역을 충분히 개척할 수 있습니다. 

저의 작은 도전이 다른 GPU Poor 개발자분들에게 조금이나마 영감과 용기가 되었으면 좋겠습니다. 앞으로 더 재밌는 실험과 오픈소스 모델들로 찾아뵙겠습니다. 감사합니다.
