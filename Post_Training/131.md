title: "NVIDIA: Speculative Decoding을 통한 RL 훈련 속도 2.5배 향상"
---

NVIDIA가 추론 최적화 기술이던 추측 해독(Speculative Decoding)을 RL 훈련 파이프라인에 통합시켜, 훈련 속도를 2.5배까지 높일 수 있음을 입증했습니다. 

RL 훈련 시간의 약 70%는 모델이 정답을 한 글자씩 생성하는 데 낭비되는데, 이 병목을 해결하려는 시도입니다. 

참고로 EAGLE-3라는 구조의 외부 초안(Draft) 네트워크를 만들어서, Qwen3-8B 모델로 실제 실험을 하고, 235B 초대형 모델 시뮬레이션 환경 실험을 진행했습니다.


1. 요리 레스토랑의 지혜 (작동 원리)

최고급 레스토랑의 메인 셰프(학습할 AI 모델)가 새로운 요리법을 연구해야 하는데, 당근을 한 땀 한 땀 직접 써느라 시간이 없습니다.

그래서 손이 빠른 보조 셰프(작은 초안 모델)를 고용합니다. 보조 셰프가 재료 3개를 한 번에 미리 썰어두면, 메인 셰프는 제대로 썰렸는지만 빠르게 눈으로 훑어보고(검증) 넘어갑니다. 

레시피 자체는 전혀 바뀌지 않으므로 요리의 맛(학습 품질)은 100% 유지됩니다.


2. 훈련 중 실시간 동기화의 마법

단순 추론 서버와 달리 RL 훈련 중에는 매 스텝 모델이 똑똑해집니다. 엔비디아는 메인 셰프가 성장할 때마다 보조 셰프도 이를 실시간으로 따라가도록 복잡한 가중치 동기화 시스템을 구현해 냈습니다.


3. 압도적인 속도 향상 지표

- 8B 모델: 단어 생성 지연 시간 1.8배 향상, 전체 학습 시간 1.41배 단축.
- 235B 초대형 모델(시뮬레이션): 단어 생성 3.5배 향상, 전체 훈련 시간 2.5배 단축. 

모델의 덩치가 크고 문장이 길어질수록 효과가 폭발적으로 증가합니다. 


4. 주의할 점: 과유불급 (한계점)

보조 셰프가 한 번에 너무 많은 재료를 썰어두면(예: 5개~7개 단어 예측) 메인 셰프가 틀린 것을 찾고 버리는 데 시간이 더 걸려 전체 속도가 느려집니다. 

논문은 한 번에 3개(k=3)를 예측할 때가 가장 완벽한 효율(Sweet Spot)을 낸다고 말합니다.


5. GRPO 전용인가요? 당장 쓸 수 있나요?

단어 생성이 필요한 DPO, SFT, 지식 증류(Distillation) 등 어디든 투입할 수 있는 만능 기법입니다.

게다가 엔비디아가 오픈소스로 공개한 NeMo RL(v0.6.0) 깃허브에 vLLM과 엮어서 이미 다 통합해 두었습니다. 설정 파일만 바꿔서 바로 실행하면 된다고 합니다.


마치며: 어떤 선택을 해야 할까?

가장 인상 깊은 점은 모델의 훈련 품질을 단 1%도 희생하지 않고 순수하게 처리량만 끌어올렸다는 점입니다. 

대규모 연산 자원이 넉넉하지 않은 우리들에게 이런 소프트웨어적 최적화는 가뭄의 단비와 같습니다. 자본력으로 밀어붙이는 독점 API에 기대지 않고, 오픈 웨이트 생태계가 이렇게 효율적으로 진화하는 모습이 참 반갑습니다.
