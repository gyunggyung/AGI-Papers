# Scaling Agent Systems: 다다익선의 함정

> **"에이전트가 많으면 무조건 좋을까? 구글과 MIT가 밝혀낸 멀티 에이전트의 과학."**

## 1. 다다익선의 함정

에이전트를 많이 쓴다고 성능이 오르는 게 아닙니다.
*   **성공 (+81%)**: 병렬 처리가 가능한 금융 분석 태스크에서는 중앙 집중형(Centralized) 에이전트가 단일 에이전트보다 80.8% 높은 성능을 보였습니다.
*   **실패 (-70%)**: 순차적인 계획이 필요한 마인크래프트(PlanCraft)에서는 오히려 성능이 70% 떨어졌습니다. 협업 오버헤드 때문입니다.

## 2. 45%의 법칙

단일 모델의 성능이 이미 **45%**를 넘는 고지능 모델이라면, 굳이 멀티 에이전트를 붙이지 마세요. 소통 비용 때문에 오히려 손해입니다.

## 3. R-squared = 0.524

이 논문은 에이전트 시스템 성능 변동성의 **52.4%**를 수학적으로 설명해냈습니다. 이제 "그냥 많이 붙여보자"는 식의 주먹구구식 설계는 끝났습니다.

## 4. 결론

순차적인 작업에는 똑똑한 **단일 에이전트(SAS)**를, 병렬 작업에는 **중앙 집중형 관리자**를 두세요. 각자도생(Independent) 에이전트는 최악의 선택입니다.
