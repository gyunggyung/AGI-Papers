알파고의 탐색을 LLM에게: 성능을 18.8% 올린 DLR (Dynamic Latent Routing)

그동안 LLM의 추론 성능을 높이려면 모델 자체를 키우거나, 프롬프트 엔지니어링(CoT 등)에 의존해야 했습니다. 

최근 베이징 항공우주대 연구진이 발표한 Dynamic Latent Routing(DLR) 아키텍처는 모델을 더 키우지 않고도, 알파고의 MCTS(몬테카를로 트리 탐색) 개념을 트랜스포머의 '잠재 공간(Latent Space)'에 이식해 추론 성능을 최대 18.8% 끌어올렸습니다.

4가지 핵심을 요약합니다.

1. 텍스트가 아닌 '생각의 흐름(잠재 상태)'을 탐색한다

기존의 에이전트 탐색 방식(ToT, MCTS)은 모델이 뱉어낸 '텍스트 문장'들을 트리 구조로 쌓아가며 좋은 답변을 찾았습니다. 당연히 텍스트를 매번 생성해야 하므로 속도가 극단적으로 느렸습니다.

반면 DLR은 트랜스포머 레이어 사이의 '잠재 공간 가중치(Hidden States)'를 트리 형태로 탐색합니다. 텍스트로 바꾸기 전인 뇌 속의 생각 상태 자체를 라우팅하므로, 토큰 생성 지연 없이 압도적으로 빠르게 최적의 추론 경로를 찾아냅니다.

2. 동적 라우터(Dynamic Router)의 의사결정

DLR은 매 토큰을 생성할 때마다 정적이지 않고 동적으로 작동합니다. 
현재 토큰에서 어떤 잠재 상태로 넘어가는 것이 최종 정답률을 높일지 라우터가 실시간으로 예측합니다. 마치 알파고가 바둑판의 다음 수(Value)를 평가하며 탐색 방향을 결정하듯, 트랜스포머 내부에서 최적의 연산 경로를 알아서 찾아가는 방식입니다.

3. 상위 레이어의 지혜를 하위 레이어로: 피드백 메커니즘

기존 트랜스포머는 입력이 들어가면 1단 레이어부터 N단 레이어까지 한 방향(Feed-forward)으로만 흘러갔습니다. 
DLR은 최종 레이어 근처의 고차원 지능(잠재 표현)을 다시 하위 레이어의 입력으로 피드백해 주는 순환 구조를 만듭니다. 모델이 스스로 "방금 내 생각의 결론이 이랬으니, 처음부터 다시 연산할 때는 이 방향을 강조해야겠다"라며 내부적으로 생각을 곱씹게 만든 것이죠.

4. 벤치마크 점수로 증명된 성능 향상

- LLaMA-2-7B 베이스 모델 대비 수학(GSM8K) 성능 18.8% 향상.
- 상식 추론 및 코드 생성 능력 12.4% 향상.
- 추가 학습이나 파라미터 증설 없이, 오직 추론 시점의 동적 라우팅만으로 달성한 결과입니다.


마치며: 하드웨어 스케일업을 넘어서는 아키텍처의 혁신

Dynamic Latent Routing은 우리에게 큰 통찰을 줍니다. 하드웨어 자원이 부족하다고 해서 반드시 성능 향상이 막히는 것은 아닙니다. 

단순히 모델 파라미터를 조 단위로 늘리거나 비싼 GPU를 더 사는 스케일업 경쟁 대신, 트랜스포머 내부의 정보 흐름(라우팅)을 영리하게 제어하고 순환시키는 소프트웨어적/아키텍처적 설계 혁신만으로도 기존 모델의 잠재력을 극한까지 끌어올릴 수 있습니다.
