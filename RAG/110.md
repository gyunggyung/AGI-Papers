# Beyond Naive RAG: 4 Papers that Redefine Agent Memory

> **"RAG는 죽지 않았다. 다만 '순진한(Naive) RAG'의 시대가 끝났을 뿐이다."**

## 1. GraphSearch (Agentic Deep Search)

기존 RAG는 멍청합니다. 질문을 던지면 비슷한 텍스트만 긁어옵니다. 하지만 **GraphSearch**는 다릅니다. 에이전트가 직접 "생각"하며 검색합니다.

*   **핵심**: 질의 분해 -> 반복 검색 -> 자기 성찰(Self-Reflection)로 이어지는 6단계 파이프라인.
*   **Dual-Channel**: 텍스트의 디테일(Semantic)과 그래프의 연결성(Relational)을 동시에 봅니다.
*   **성능**: MuSiQue 벤치마크(복잡한 질문)에서 정답 포함률(SubEM)이 35.00%에서 **51.00%**로 수직 상승했습니다.

## 2. S-RAG (Structured RAG)

"모든 호텔 중 평점이 가장 높은 곳의 예약 가능 여부는?" 이런 집계형 질문(Aggregation)은 벡터 DB의 천적입니다.

*   **해법**: 비정형 문서를 아예 **SQL 테이블**로 변환해버립니다. LLM이 텍스트를 읽는 게 아니라, SQL 쿼리를 짜서 DB를 분석합니다.
*   **성능**: HOTELS 데이터셋에서 기존 VectorRAG의 재현율(Recall)은 0.352였으나, S-RAG는 **0.845**를 기록했습니다.

## 3. xMemory (Beyond RAG for Agent Memory)

에이전트와의 긴 대화를 RAG로 저장하면 중복된 데이터만 쌓입니다. 에이전트 메모리는 거대한 위키피디아가 아니라, 흐르는 '스트림'이어야 합니다.

*   **구조**: 메시지 -> 에피소드 -> 시맨틱 -> 테마의 **4계층 구조**.
*   **효율**: LoCoMo 벤치마크에서 토큰 사용량을 29% 줄이면서도 F1 점수는 36.42에서 **43.98**로 올렸습니다.

## 4. Graph-based Agent Memory Survey

이 모든 흐름을 관통하는 키워드는 **그래프(Graph)**입니다. 메모리는 단순 저장이 아니라 추출 -> 저장 -> 검색 -> 진화라는 생명주기를 가집니다.

*   **자기 진화(Self-Evolving)**: 쓰면 쓸수록 메모리의 구조가 스스로 최적화되고 강화됩니다.

## 결론

2026년의 에이전트는 로그를 쌓는 게 아니라, 경험을 구조화하여 자신의 '세계관'을 구축해야 합니다. **"구조화된 연결(Structured Connection)"**이 핵심입니다.
