MiniMax M2 시리즈 기술 보고서가 공개됐습니다. 최근 어떤 부분을 읽지 않을지를 고민하며 어텐션을 깎아내는 트렌드와 달리 MiniMax는 Full Attention으로 문맥을 유지합니다. 그리고 M2.7부터 모델이 스스로 오류를 수정하며 발전하는 자가 진화를 시작했습니다. 

229.9B A9.8B 모델이 Claude 4.6, GPT 5.4, Gemini 3.1 Pro 같은 프론티어 모델들과 대등하게 싸우는 핵심을 두 축으로 정리합니다.

1. 첫 번째 축: 깎아내지 않은 풀 어텐션과 지우지 않는 생각

요즘 전 세계 AI 랩들의 최대 고민은 거대한 컨텍스트를 처리할 때 GPU 메모리가 터져나간다는 것입니다. 이를 해결하기 위해 많은 기업이 어텐션 창을 슬라이딩(SWA)하거나 깎아내는 효율적 어텐션 트렌드에 매진합니다.

하지만 미니맥스 연구진은 경고합니다.

 - 표준 벤치마크의 착시: SWA 같은 하이브리드 어텐션을 섞어 써도, 일반적인 단답형 시험에서는 풀 어텐션과 점수 차이가 거의 없었습니다. 많은 기업이 이 점수에 속아 하이브리드 어텐션 모델을 서빙합니다.

 - 실제 에이전트 환경에서의 붕괴: 하지만 문맥이 32K를 넘어가고, 복잡한 추론이 필요한 진짜 에이전트 작업에 돌입하자 SWA 변형 모델들의 성능이 처참하게 무너졌습니다. 초장문 안에서 단서들을 서로 연결하는 멀티홉(Multi-hop) 추론 능력이 문맥 차단 한계 때문에 작동하지 않은 것이죠.

미니맥스는 의미없는 벤치마크 점수보다 진짜 지능을 위해 Full Attention 구조를 굳건히 밀어붙였습니다.

이 뚝심은 인터리브드 씽킹(Interleaved Thinking)이라는 인지 루프로 이어집니다. 도구를 사용할 때, 메모리를 아끼기 위해 이전 턴의 생각 블록들을 컨텍스트에서 지워버리는 기존 방식과 달리, M2는 과거의 모든 생각 흔적을 192K 문맥 전체에 온전히 누적합니다.

덕분에 코딩을 하다가 버그나 에러 로그를 만났을 때, "내가 3턴 전에 A라는 가설을 세우고 라이브러리를 가져왔는데, 이 에러가 난 걸 보니 가설 A가 틀렸구나. B 계획으로 선회하자"라는 인간 개발자 특유의 Plan-Act-Reflect(계획-행동-반성) 루프가 완벽히 작동하게 됩니다. 완벽하게 실패를 기록하고 추적할 수 있는 뇌 구조(Full Attention)가 뒷받침되기에 점진적으로 오류를 잡을 수 있는 것입니다.


2. 두 번째 축: 자가 진화(Self-Evolution)의 시작

"개발을 완벽히 할 수 있어야 자기 오류도 수정할 수 있다."

이 난제를 풀기 위해, 미니맥스는 모델을 먼저 인간이 구축한 고정밀 샌드박스에서 혹독하게 훈련시켰습니다. 수만 개의 코드를 고치고, 가상 환경에서 앱을 배포하고 검증받으며 에이전트 고유의 개발 실력을 전문 개발자 급으로 끌어올렸습니다.

기초 체력이 완성된 M2.7 시점에 이르러 마침내 자가 진화의 문이 열렸습니다.

사람이 목표를 설정하면, M2.7은 격리된 가상 작업 공간(Agent Harness) 안에서 혼자 구동을 시작합니다. 스스로 인프라의 실패 로그와 훈련 메트릭을 분석하여, 자신의 에이전트 스캐폴드나 훈련 구성 파일, 코드를 스스로 수정하고 디버깅하는 100라운드의 완전 자율 최적화 사이클을 수행한 것입니다.

이 과정에서 인간 연구원도 생각지 못한 무한 루프 탐지 메커니즘 코드를 모델이 스스로 개발해 자기 몸에 이식했고, 사내 평가 기준 30%의 성능 향상을 스스로 달성해 냈습니다. 사람이 아닌 AI가 직접 코드를 리팩토링해 성능을 올린 최초의 공개 실전 사례 중 하나입니다.


3. 40배 빠른 학습을 가능케 한 인프라의 마법: Forge

에이전트 강화학습은 문맥이 너무 길어 기존 시스템으론 감당이 안 되기에, 자체 개발한 Forge 인프라로 이를 해결했습니다.

 - Prefix Tree Merging (접두사 트리 병합): 멀티턴 대화나 에이전트 조작 시 앞부분의 겹치는 긴 문맥들을 트리 구조로 묶어 전방 연산(Forward Pass)을 딱 한 번만 수행합니다. 수학적 오차는 0%이면서 학습 속도를 최대 40배 가속하고 메모리를 극적으로 아꼈습니다.

 - 속도/효율성 보상: 단순 결과의 정답 유무뿐만 아니라, 실제 구동 시간을 보상 함수에 산입했습니다. 에이전트가 지연 시간을 최소화하기 위해 가장 빠르고 효율적인 도구 호출 경로(병렬 처리 등)를 스스로 찾아내도록 유도한 공학적 묘미입니다.


4. 숫자로 증명하는 압도적 성과

단 10B의 활성화 파라미터로 이뤄낸 벤치마크 점수는 이 설계 철학이 완벽히 맞았음을 수치로 증명합니다.

 - SWE-bench Pro: 56.2점 기록 (Claude Sonnet 4.6의 57.7점, GPT 5.4의 57.2점과 동등 수준이며 Gemini 3.1 Pro의 54.2점을 제침).

 - VIBE-Pro (풀스택 앱 개발): 55.6점 기록 (Claude Opus 4.6과 공동 1위).

 - MLE Bench Lite (자율 머신러닝 엔지니어링): 66.6% 메달 획득률 기록 (Gemini 3.1 Pro와 완벽한 동률). AI가 스스로 AI 파이프라인을 다루고 최적화할 수 있음을 공인받은 지표입니다.

 - AIME 2026 (수학): 94.2점 기록 (Claude Opus 4.6의 92.5점, Gemini 3.1 Pro의 88.7점 돌파).


마치며: 결국 기본기와 시스템 최적화입니다.

완벽해서 스스로를 고치는 게 아니라, 안전하게 실패할 수 있는 격리된 놀이터(Docker 샌드박스)를 주고, 실패를 온전히 기억할 수 있는 단단한 뇌 구조(Full Attention)를 쥐여주었기에 고치면서 완벽해지는 지능의 선순환이 가능했습니다.

인프라 비용을 아끼겠다고 어텐션을 성급하게 깎아냈다면, M2.7은 장기 디버깅 로그의 무게를 견디지 못하고 자가 진화 근처에도 가지 못했을 것입니다. 화려한 수학적 변형보다 한정된 하드웨어 자원을 알뜰하게 나눠 쓰고 영리하게 구조화한 공학적 뚝심이 혁신을 만들었습니다.

거대 모델의 환상에만 속지 말고, 다시 기본기로 돌아가야 할 때입니다. (물론 이것도 너무 큽니다)
