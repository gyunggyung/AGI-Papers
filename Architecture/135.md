title: "HOLA (Hippocampal Linear Attention): 순환 모델의 구조적 건망증 해결"
---

이론상 무한한 문맥을 다룬다는 순환 모델(Mamba, GDN 등)들은 압축으로 인한 구조적인 건망증을 가지고 있습니다. 뇌 구조(해마와 대뇌피질)에서 영감을 얻어 이 문제를 해결하려는 HOLA(Hippocampal Linear Attention)를 소개합니다.

1. 압축의 저주: 대뇌피질만 있는 AI

기존 선형 모델은 메모리를 아끼기 위해 수만 글자의 문맥을 고정된 작은 상자(256 x 256 행렬)에 계속 욱여넣으며 압축합니다. 

문맥의 흐름이나 규칙을 파악하는 대뇌피질의 역할은 잘 수행하지만, "비밀번호는 1234" 같은 작고 구체적인 팩트 알갱이들은 새로운 정보에 덮어쓰여 뭉개져 버립니다. 

긴 글 속에서 특정 정보를 찾는 바늘 찾기(Needle-in-a-haystack) 태스크에서 정답률이 바닥을 치는 이유입니다.

2. 해마의 탄생: "가장 가끔 나오는 것만 딱 64개 기억해!"

HOLA는 뇌과학의 상호보완 학습 시스템(CLS) 이론을 빌려왔습니다. 일반적인 맥락은 기존 모델(대뇌피질)이 압축하게 두고, 모델 내부에 딱 64칸짜리 무유실 해마 캐시를 추가했습니다. 

64칸인 이유는, 기존 256 x 256 크기의 기억 행렬이 실제로 담을 수 있는 정보의 용량(유효 Rank)이 딱 64로 제한되어 있기 때문입니다.

여기서 공학적 묘수가 하나 더 들어갑니다. 문장을 한 단어씩 읽으면 GPU 연산 효율이 떨어지기 때문에, 모델은 기본적으로 256개 단어(청크)씩 묶어서 처리합니다. 

이때 지금 처리 중인 현재 블록(256개 단어)은 점수와 상관없이 전부 임시 캐시에 통째로 들고 있습니다. 눈앞의 문맥과 인과관계는 완벽하게 다 보면서 연산 속도도 챙기는 셈입니다.

그리고 이 256개 블록이 지나가고 과거의 기억이 되는 순간, 모델이 예측에 실패해서 내부 상태값을 가장 크게 뒤흔든 가장 가끔 나오는 토큰(Surprise 점수)들만 거리에 상관없이 박제합니다. 

문맥상 뻔하게 예측되는 단어는 버리고, 가끔 튀어나오는 고유명사나 핵심 수치 같은 알갱이만 사진처럼 찍어두는 겁니다.

3. 쓰레기통(Null Sink)과 송곳(RMSNorm)의 미학

기존 선형 모델의 캐시는 기억을 꺼낼 때 흐릿한 평균값으로 꺼냅니다. 시험 문제를 풀 때 "대충 1번과 2번이 섞인 느낌"으로 답을 적는 꼴이죠. 

하지만 HOLA는 기억을 읽어올 때 모델 업데이트 경로와 분리된 RMSNorm을 적용하여, 어텐션의 분포를 선명하게 깎아내어 명확한 정답만 꺼냅니다. 

만약 내가 아예 배운 적 없는 엉뚱한 질문이 나오면 어떻게 될까요? 즉, 캐시 주머니(64개)를 아무리 뒤져도 힌트가 없는 상황입니다. 

잘못된 정보를 인용해서 할루시네이션이 발생하기 쉬운 상황이지만, HOLA는 캐시 맨 첫 칸에 Null Sink(공백의 쓰레기통)라는 더미 토큰을 하나 심어두었습니다. 

힌트가 없을 땐 어텐션을 이 쓰레기통(Null Sink) 토큰에 몰아주어 캐시의 영향력을 완전히 청소해 버립니다. 

잘못된 캐시 정보에 방해받지 않고, 모델 원래의 실력(순환 상태 지식)만으로 안전하게 정답을 고르게 만드는 영리한 장치입니다. 

4. 압도적인 숫자: 지능은 오르고 비용은 그대로

340M 소형 모델을 문맥 길이 2k로 15B 토큰으로 밑바닥부터 학습시킨 결과입니다.

 - 언어 모델링의 진화: 기존 모델 대비 Perplexity를 16.1%나 급감시키며 Transformer++(26.88)보다 낮은 22.92를 기록해, 풀 어텐션 모델의 지능까지 이겨버렸습니다. (낮을수록 좋음)

 - 32배 긴 바늘 찾기: 문맥을 32k로 늘리자 기존 선형 모델은 정답률이 14%로 추락했지만, HOLA는 58%를 버텨내며 압도적인 롱컨텍스트 생존력을 보여주었습니다.

 - 공짜에 가까운 지능: 이 모든 걸 해내는데 추가된 파라미터는 전체 모델의 0.004%에 불과하며, 추가 GPU 메모리 점유율도 고작 5% 남짓입니다. 심지어 문맥이 무한히 길어져도 캐시가 64개로 고정되어 있어 메모리 증가율은 0(Flat)입니다.

5. 영리함 뒤에 숨은 명확한 한계

물론 이 모델이 모든 면에서 완벽한 만능은 아닙니다. 구조적 특성상 2가지 명확한 한계가 존재합니다.

- 텍스트 맹신 금지: 복잡한 정보 추출 테스트에서는 기존 선형 모델들을 추월한 것은 맞지만, 모든 토큰을 무손실로 처음부터 끝까지 다 쥐고 있는 오리지널 풀 어텐션 모델의 절대적인 성능에는 아직 미치지 못합니다.

- 촘촘한 바늘 앞에서의 무력함: 해마의 방 크기가 딱 64칸으로 제한되어 있습니다. 즉, 수만 단어짜리 긴 문서 전체에 반드시 암기해야 하는 핵심 정보(바늘)가 수백 개 이상 촘촘하게 박혀 있는 극단적인 상황이나 훨씬 더 긴 롱컨텍스트 상황에서는 정보를 놓칠 수밖에 없습니다.

마치며

뇌 구조에서 영감을 받아서 나온 아키텍처들이 계속 나오고 있습니다. 물론 실제 뇌의 동작 방식과는 다르고, 그 효율성과 능력을 미치지는 못합니다. 

순환 모델 진영이 아직은 압도적인 성과를 보여주지는 못했지만, 계속 발전하고 있습니다. 추후에는 이 영역에서도 더 멋진 성과가 나오고, 이를 기반으로 한 Qwen3.5나 Nemotron3를 넘는 하이브리드 모델들도 많이 나오면 좋겠습니다.
