# 트랜스포머의 성능, RNN의 가벼움을: Memory Caching (MC)

요즘 LLM의 가장 큰 고민거리인 '기억력(Memory)' 딜레마를 해결하기 위해, Google 리서치와 Cornell University 연구진이 발표한 'Memory Caching(MC)' 논문은 현실적인 절충안을 제시합니다.

핵심 구현 디테일과 이것이 왜 중요한지 4가지로 정리했습니다.

### 1. 시험 공부의 비유: 요약본 쪼개기

트랜스포머는 시험장에 전공 서적 수백 권을 다 들고 가는 학생입니다. O(L²)의 복잡도를 가지죠. 모든 내용을 찾을 수 있지만 시간이 엄청 걸립니다. 

반면 기존 RNN은 수백 권의 책을 단 한 장의 깜지로 억지로 압축해 가는 학생입니다. O(L)로 빠르지만 디테일은 다 날아갑니다.

MC는 시퀀스를 여러 '세그먼트'로 쪼갭니다. 챕터별 요약본(캐시된 메모리 상태)을 따로 만들어 두는 방식입니다. 

문제를 풀 때 현재의 기억뿐만 아니라, 과거에 저장해둔 요약본들을 같이 꺼내어 봅니다. 트랜스포머와 RNN 사이에서 타협점인 O(NL) 복잡도를 만들어낸 것입니다.

### 2. 숫자로 증명된 압도적 가성비

 - **정보 검색(Needle-in-a-Haystack, 16K)**: 기존 DLA 모델은 16K 길이 문서 검색 정답률이 44.0%에 불과했습니다. 하지만 GRM(Gated Residual Memory) 방식을 적용하자 정답률이 82.4%로 거의 2배 폭등하며 트랜스포머(79.8%)를 이겨버렸습니다.

 - **속도(Throughput) 역전**: 문맥 길이가 16K를 넘어가는 순간, 트랜스포머의 초당 처리 속도는 수직으로 낙하합니다. 반면 선택적 라우팅(SSC)을 적용한 MC 모델은 32K가 넘어가도 35K~39K t/s로 평탄하게 속도를 방어하며 트랜스포머를 가볍게 추월합니다.

 - **MQAR(다중 쿼리 연관 검색)**: 파라미터 차원당 최고 성능을 기록하며 기존 SOTA 모델인 Atlas를 능가했습니다.

### 3. 중요한 핵심 4가지

단순히 "메모리를 쪼개서 합친다" 정도로 요약하고 넘어가기엔, 이 논문이 던지는 엔지니어링적 통찰이 너무 큽니다.

 - **하이브리드 아키텍처의 비밀 해제**: 요즘 유행하는 RNN과 Attention을 교차로 쌓는 하이브리드 구조를 수학적으로 뜯어보면, 결국 '메모리 캐싱'의 특수한 케이스입니다. 하이브리드 모델이 왜 잘 작동하는지에 대한 명확한 수학적 직관을 제공합니다.

 - **세그먼트 분할의 딜레마 (Constant vs Logarithmic)**: 그냥 쪼개는 게 아닙니다. 동일한 크기로 자를지, 로그 단위로 자를지 고민해야 합니다. 연산 효율은 로그 방식이 좋지만, 긴 부분 시퀀스가 메모리 오버플로우를 유발하고 성능이 떨어져 논문은 일정한 크기(Constant) 방식의 손을 들어줍니다.

 - **Deep vs Linear 메모리의 차이**: 베이스가 되는 메모리가 선형 행렬인지 비선형 신경망(Deep)인지에 따라 작동 방식이 다릅니다. 깊은 메모리에서는 파라미터 자체를 섞어 매 토큰마다 동적인 검색 함수를 만드는 'Memory Soup' 기법이 진짜 위력을 발휘합니다.

 - **사전 학습 모델 재활용 (Post-Training)**: 바닥부터 새로 학습할 필요가 없습니다. 이미 학습이 끝난 모델에 가중치 업데이트 없이 이동 평균 캐시만 달아서 추론 단계에 바로 적용해 문맥 길이를 확장할 수 있습니다.

### 4. 현실적인 한계점 (The Catch)

이 논문도 확실한 한계가 있습니다.

 - **스케일링 검증 부족**: 가장 큰 실험 모델이 고작 1.3B 파라미터입니다. 현재 실무에서 다루는 7B 이상의 모델이나 테라바이트급 데이터 환경에서도 이 효율이 붕괴되지 않을지는 미지수입니다.

 - **하드코어 검색은 여전히 트랜스포머**: 긴 문맥에서의 UUID 검색 같은 극단적인 태스크에서는 여전히 과거를 전부 뒤지는 트랜스포머가 우위를 점하고 있습니다.

 - **순수 RNN보단 무겁다**: 체크포인트를 캐싱할지, 세그먼트마다 독립된 메모리를 쓸지 설계적 고민이 필요하며, 아무리 라우터로 최적화해도 완벽한 선형 복잡도를 갖는 순수 RNN보다는 메모리와 연산 오버헤드가 발생합니다.

### 💡 마치며: 롱컨텍스트를 위한 새로운 무기

무작정 파라미터를 키우고 컨텍스트 윈도우를 1M, 2M 단위로 늘려버리는 건 막대한 컴퓨팅 파워를 가진 빅테크나 가능한 일입니다. 

한정된 자원으로 로컬 모델을 돌려야 하는 우리들에게 "필요한 덩어리만 묶어서 캐싱한다"는 효율적인 접근은 매우 실용적인 무기입니다.

단순한 아이디어로 병목을 뚫어내는 이런 연구들이, 앞으로 우리가 다룰 오픈소스 소형 모델들에 빠르게 이식되기를 기대합니다.
