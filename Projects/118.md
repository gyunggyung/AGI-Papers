# 3개월 9개 프로젝트 회고: GPU Poor의 여정

> **RAM 16GB, i5 CPU가 탑재된 평범한 윈도우 노트북에서 9개의 프로젝트와 2가지 아카이브를 만들었습니다.** 가장 흔하고 제한적인 환경에서 과연 어디까지 가능할지 직접 확인해 본 기록입니다.

## 🎯 1. Models & Architecture (모델 학습 및 아키텍처)

*   **LFM2-KoEn-Tuning**: LFM2 1.2B를 튜닝해 구글 Gemma-3-4B와 알리바바 Qwen3-4B의 번역 성능을 뛰어넘은 한영 번역 특화 모델. (Liquid AI 공식 Laptop Examples 등재)
*   **Korean Linux Colab (HybriKo-117M)**: 117.8M 하이브리드(RNN+Attention) 아키텍처를 바닥부터 훈련시켜 한국어를 리눅스 명령어로 번역하도록 만든 프로젝트.
*   **Open-Yaongi (PoC)**: 52-Layer 4B (0.6B Active) MoE 아키텍처의 PoC 구현체. 단일 H100으로 학습 가능하게 설정.
*   **Liquid-ASA**: 모델 재학습 없이 추론 시점에서 도구 호출(Tool-Calling) 능력을 향상시키는 활성화 스티어링 어댑터 파이프라인.

## 🧠 2. Agents & Inference (로컬 에이전트 및 추론)

*   **Tiny MoA (Mixture of Agents)**: VRAM 0MB(CPU Only) 환경에서 1.2B Brain 모델과 0.6B Specialist, 90M Tool-Call 모델들이 협업하는 초경량 에이전트 시스템.
*   **Agent.cpp**: Tiny MoA 전용으로 Llama.cpp 기반 캐싱을 개선하여 바이브 코딩으로 구축한 추론 엔진.
*   **Gemini-Claw**: API 기반 LLM의 한계와 통제 불가능성을 실험하며, CoWork 기능을 무료로 구현해본 CLI 기반 툴.

## 🛠️ 3. Practical Tools (실용적 연구/업무 보조 툴)

*   **LFM-Scholar**: LFM2 2.6B 및 LFM2.5 1.2B 모델을 미세 조정해서 논문을 검색하고 Related Work 섹션을 통째로 작성해 주는 로컬 연구 보조 도구.
*   **docling-translate**: IBM Docling을 기반으로 논문의 수식과 레이아웃을 훼손하지 않고 로컬 LLM으로 번역하는 문서 번역기.

## 📚 4. Study & Archive (데이터셋 및 연구 아카이브)

*   **LLM-KO-Datasets**: Pre/Mid/Post-training에 필요한 고품질 한국어 및 다국어 데이터셋 구축 전략과 자료를 모아둔 저장소.
*   **AGI-Papers**: LinkedIn에 쓴 글을 8가지 핵심 주제로 분류하여 정리한 논문 리뷰 및 인사이트 아카이브.

## 💡 기술적 확신, 그리고 새로운 갈증

태스크를 충분히 잘게 쪼갤 수만 있다면, 거대한 최신 API 모델과 파라미터 90M~3B 규모의 sLM 간의 결과물 차이는 미미해집니다. 고작 RAM 16GB의 로컬 환경에서도 충분히 의미 있는 서비스가 완벽한 통제(Control) 하에 돌아갈 수 있다는 가능성을 보았습니다.

동시에 아직도 거대 LLM이 필요한 부분도 명확히 알게 됐습니다. sLM 수준에서는 아직 스스로 작업을 분해하거나 복잡한 문제를 해결하는 것이 거의 불가능합니다. 반면, 거대 LLM은 Task Decomposition, Planning을 아주 훌륭하게 해냅니다.

하지만 역설적이게도 혼자서 이 모든 걸 만들다 보니 뼈저리게 느낀 갈증이 하나 있습니다.

> **"나는 지금 진짜 문제를 풀고 있는가, 아니면 가상의 문제를 만들어내고 있는가?"**

기술적 병목을 해결하는 즐거움도 컸지만, 이제는 상상 속의 시나리오가 아닌 사람들에게 실질적인 도움을 주는 진짜 문제를 풀고 싶습니다.
