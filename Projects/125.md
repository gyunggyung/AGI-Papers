변호사 시험 만점을 받았습니다. 제15회 변호사 시험 선택형 150문제 중에 150개를 전부 맞혔습니다.

참고로, 엘박스 실험 결과에 따르면 150문제 중 GPT-5.4는 94개, Claude Opus 4.6은 98개, Gemini 2.5 Pro는 110개를 맞혔다고 합니다. 엘박스 AI는 141개, 슈퍼로이어는 150개를 맞혔다고 합니다. 그리고 제15회 변호사시험 전국 수석은 144개를 맞혔다고 합니다. 

리걸테크 기업들이 막대한 자본과 수백만 개의 독점 판례 DB를 활용해 만점에 도달할 때, 저는 공개 데이터만 가지고 만든 결과입니다. 

한국 변호사 시험은 프론티어 모델들도 잘 못 푸는 영역입니다. 그럼에도 각 문제 해결에 필요한 법률적 근거를 컨텍스트로 준다면 무조건 다 맞을 수 있다고 생각했습니다.

1. Codex GPT-5.5 기반 법률 컨텍스트 구축

문항별 판례, 법령, 하위규범을 쪼개고 함정 포인트를 요약한 문제별 법률 컨텍스트를 구축했습니다. 데이터 충돌이나 누락 위험을 줄이기 위해 GPT가 자동으로 내용 검토를 하며 근거를 검증합니다. 

2. Antigravity Gemini 3.1 Pro의 추론 제어

모델에게 오직 제공된 근거 문서만 참고해서 문제를 풀라고 지시했습니다. '지문별 O/X 판단 -> 목표 지문 집합 도출 -> 선택지 조합 대조 -> 최종 검산'으로 이어지는 4단계 프로세스를 강제하여 모델의 단순 논리 오류를 차단했습니다. (참고로 /goal 명령을 쓰면 150 문제를 한 번에 풀어줍니다)

3. 시행착오

법률 컨텍스트 구축에 수십 시간이 소요됐습니다. 한 번 150문제에 대한 법률 근거를 검색하고 찾을 때 20시간 이상이 소요됩니다. 

해당 컨텍스트들이 올바른 법령과 판례를 잘 가져왔는지 링크를 검증하고, 해당 문제에 적합한 컨텍스트가 맞는지 검증하고 하는데 10시간 이상이 소요됩니다. 이 과정을 5번 반복했습니다. (병렬로 처리하면 Codex가 빠르게 죽습니다)

4. 모델의 한계

Gemini 3.5 Flash 모델은 정답을 맞혀놓고도 오답을 선택하는 이상한 문제가 있었습니다. 그리고 GLM5.2는 컨텍스트를 줘도 42% 정도의 정답률을 보였습니다. 31B 이하의 오픈 모델들은 다지선다 자체도 못했습니다. 

gemma-4-31B-it 모델을 1회에서 14회 변호사 시험과, 합성 데이터 2천 문제를 학습 시켰을 때 50% 정도의 정답률을 보였습니다. Qwen3.6-27B는 학습을 해도 해당 성능도 나오지 않았죠.

그리고 10B 이하의 모델들은 한국 법, 판례, 행정 데이터인 Legalize-KR의 데이터들을 Pre-training해도 CPT를 해도 찍는 수준의 정답률을 보였습니다. 우리나라의 변호사 시험이 참 어려운 것 같습니다.
