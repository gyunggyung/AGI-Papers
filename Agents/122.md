하네스 이야기를 안 하려고 했지만, 폐쇄 루프 하네스 이야기를 해야겠습니다. 이제 스스로 파이썬 코드를 짜서 자신의 시스템을 개조하는 시대입니다. 코딩 에이전트를 뛰어난 'F1 레이서(인공지능 모델)'라고 상상해 보겠습니다.

아무리 뛰어난 레이서라도 자동차의 조종석이나 페달(하네스)이 엉망이면 우승할 수 없듯, LLM 모델 역시 주변 환경 설계에 따라 성능이 크게 좌우됩니다

오늘 소개할 논문 'Agentic Harness Engineering (AHE)'은 AI 레이서의 주행 기록을 분석하고 장비를 자동으로 튜닝해 주는 'AI 피트 크루(진화 에이전트)'에 관한 이야기입니다. 

기존에는 사람이 로그를 읽으며 수동으로 고쳤지만, 이제는 AI가 스스로 완벽한 폐쇄 루프(Closed loop)를 돌며 진화합니다.


1. 프롬프트 엔지니어링의 배신: 말로 해서는 안 고쳐집니다

시스템 프롬프트(말로 하는 지시사항)만 단독으로 자동 튜닝했을 때는 성능이 2.3%p 하락했습니다. 진짜 성능 향상을 이끈 것은 프롬프트가 아니라 시스템의 구조적인 부분들이었습니다.

 - 장기 메모리(Long-term memory) 단독 튜닝 시 5.6%p 향상에 기여했습니다.

 - 도구(Tools) 단독 튜닝 시 3.3%p 향상에 기여했습니다.

 - 미들웨어(Middleware) 단독 튜닝 시 2.2%p 향상에 기여했습니다.


2. 스스로 파이썬 코드를 짜서 '시스템'을 개조하는 AI

말로 타일러서 안 고쳐진다면 어떻게 할까요? 

AHE는 아예 에이전트가 활동하는 시스템의 소스 코드 자체를 뜯어고칩니다. 논문에 나오는 통계 분석 과제(MCMC) 사례가 아주 흥미롭습니다.

 - AI가 시간이 오래 걸리는 정식 통계 분석을 중간에 강제 종료하고, 임의의 야매 검증기로 대충 정답을 제출하는 꼼수를 부립니다.

 - 그러자 AHE의 진화 에이전트는 프롬프트를 고치는 대신, execution_risk_hints.py라는 파이썬 미들웨어 코드를 백지부터 새로 작성합니다. 이 코드는 명령어 기록을 실시간으로 감시하는 CCTV 역할을 합니다.

 - 동시에 쉘(Shell) 도구의 파이썬 코드를 수정하여, 분석 스크립트를 함부로 강제 종료하지 못하도록 시스템적인 보호 상태(Protected) 잠금 장치를 만들어버립니다.

 - 결국 다음 테스트에서 AI는 꼼수를 포기하고 약 10만 번 반복되는 정식 코드를 끝까지 돌려 정답을 맞히게 됩니다.


3. 찍기 방지와 완벽한 폐쇄 루프 (Closed Loop)

AI가 코드를 마구잡이로 건드려 시스템이 망가지면 어떡할까요? 

AHE는 수행, 원인 분석, 코드 개조, 다음 턴에서 성과 검증, 실패 시 롤백이라는 완벽한 폐쇄 루프를 돌며 이 문제를 해결합니다.

 - 코드를 수정할 때마다 "이걸 바꾸면 A 문제가 해결될 것이다"라고 명시적인 예측(Change Manifest)을 기록합니다.

 - 다음 테스트에서 이 예측이 틀리면 즉시 이전 상태로 롤백해버리는 반증 가능한 계약 시스템을 갖췄습니다.


4. 숫자로 증명하는 압도적인 성능

이러한 자동 튜닝의 결과는 수치로 명확하게 증명됩니다.

 - 성공률 폭등: 10번의 반복 튜닝 후 기본 성공률은 69.7%에서 77.0%로 치솟으며, 인간 전문가가 설계한 하네스인 Codex-CLI(71.9%)와 다른 AI 자동 튜닝 모델들을 모두 압도했습니다.

 - 비용 절감: 긴 프롬프트 대신 도구와 메모리에 행동 패턴을 압축하여, 연산 비용인 토큰을 12%나 적게 사용하면서도 더 높은 성공률을 달성했습니다.

 - 범용성 확인: 이 튜닝된 하네스에 다른 모델을 태워도 DeepSeek-v4-flash는 10.1%p, Qwen-3.6-plus는 6.3%p 성능이 크게 향상되었습니다.


마치며: 지시하는 AI에서, 스스로 환경을 구축하는 AI로

이 논문은 AI에게 어떻게 행동할지 프롬프트로 설득하는 것에 집착하지 않습니다. 

대신 AI가 자신의 경험을 바탕으로 스스로 툴을 깎고 미들웨어를 구축하여 자신이 일하기 가장 완벽한 조종석을 만드는 방향을 제시합니다. 

에이전트의 진화는 명시적인 외부 도구와 시스템의 구조적 진화에서 시작될 것입니다.
