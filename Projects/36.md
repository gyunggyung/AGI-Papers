직접 설계한 117M 모델이 한국어를 리눅스 명령어로 바꿔줍니다. 요즘 유행하는 100B 모델이 아니라 100M 급 초경량 모델입니다. 아키텍처 설계부터 Pre-training, 리눅스 명령어 SFT까지 - From Scratch로 이 작업에 최적화된 모델을 Hongseok Lee님과 만들었습니다. Yaongi/HybriKo-117M-LinuxFC-SFT-v2 입니다. 그리고 이 모델을 이용해서 바로 사용할 수 있게 🐧 Korean Linux Colab 프로젝트를 만들었습니다.

왜 이런 걸 만들었냐면요.

2010년대 중반, 고등학생 시절 리눅스마스터 자격증을 따며 터미널과 처음 씨름했던 기억이 납니다. 그때나 지금이나, 처음 마주하는 검은 화면과 수많은 명령어는 누구에게나 높은 진입장벽이죠.

최근 Claude Code 같은 코딩 에이전트들이 각광받는 이유도 여기에 있다고 봅니다. 복잡한 명령어를 외우는 대신, 자연어로 소통하며 개발에만 집중하고 싶은 니즈가 커진 것이죠.

여기서 힌트를 얻었습니다. "단순히 리눅스 명령어를 변환하는 거라면, 굳이 무거운 100B급 거대 모델을 돌릴 필요가 있을까?" 117M 초경량 모델로도 충분히 가능하고, 누구나 무료로 가볍게 쓸 수 있다면 더 효율적일 테니까요.

그래서 한국어로 말하면 리눅스가 알아듣는 도구를 만들었습니다.

"현재 폴더에 뭐 있어?" → ls -la
"test.txt 내용 보여줘" → cat test.txt
"구글에 핑 날려봐" → ping -c 4 google.com

🎯 핵심 가치
Colab에서 1분 만에 실행. 설치도 필요 없습니다. 리눅스를 처음 배우는 분들이 명령어를 외우기 전에 먼저 "이게 뭘 하는 건지" 체험할 수 있습니다.

📊 결과
- 92% 정확도 (12개 테스트 중 11개 성공)
- 21개 명령어 지원 (ls, cd, mkdir, rm, cp, mv, find, cat, grep, head, tail, wc, ps, df, du, top, ping, curl, chmod, tar 등)

⚠️ 한계
117M 소형 모델이라 완벽하지는 않습니다. 가끔 파일명을 살짝 다르게 생성하기도 합니다. 하지만 "어떤 명령어를 써야 할지"는 잘 맞추니까, 파라미터는 코드로 보정하면 실용적으로 충분합니다.

💡 바라는 점
개발을 시작하는 분들이 터미널 앞에서 막막해하지 않았으면 좋겠습니다. 이 도구가 그 첫 걸음을 조금이라도 가볍게 만들어줄 수 있다면 좋겠습니다.

🚀 확장
다음 버전에서는 코딩 에이전트에 통합해서 Claude Code처럼 직접 명령을 실행하게 만들려고 합니다. 지금은 117M의 한계가 명확하지만, 더 큰 모델로 발전시킬 예정입니다.

Apache 2.0으로 공개합니다.
