이제 RAG에서 구식 Vector DB를 버리고, 지식만 전문으로 암기하는 소형 비서 sLM 모델을 따로 만들어 LLM 옆에 두면 어떨까요? 현존 최고라는 Graph RAG조차 2배 이상 압도하는 성능을 낸다면 어떨까요?

MIT, NUS, Liquid AI 등 연구진이 발표한 최신 논문 "MEMO: Memory as a Model"은 새로운 방법을 제시합니다.

1. 도서관 사서 대신 요약 비서 AI 고용하기: Memory as a Model

기존 RAG가 도서관 사서(벡터 DB)에게 "관련 문서 조각 좀 찾아서 내 책상(컨텍스트 창)에 쌓아줘"라고 부탁하는 방식이라면, MEMO는 머리는 작지만 기억력이 엄청나게 좋은 소형 비서(MEMORY 모델, 1.5B~14B 체급)를 따로 학습시켜 옆에 앉혀두는 방식입니다.

메인 브레인 역할을 하는 거대 LLM(EXECUTIVE 모델, 예: Qwen 32B, Gemini 3 Flash)은 직접 외부 DB를 뒤지는 대신, 이 소형 비서 AI에게 자연어로 꼬리 질문을 던지며 지식을 인터뷰하듯 캐냅니다. 메인 모델의 내부 가중치는 건드리지 않고 얼려두기 때문에 파멸적 망각 위험이 없습니다.


2. 기적의 5단계 천재 학습법으로 이식하는 지식

이 소형 비서에게 지식을 가르칠 땐 무작정 원문을 읽히지 않습니다. 스타 강사 LLM(Qwen 32B)을 시켜 문서를 촘촘한 QA 형태의 문제집으로 정제합니다.

 1단계 (Fact Extraction): 문단의 표면적 팩트와 숨겨진 맥락을 샅샅이 문제와 답으로 쪼갭니다.

 2단계 (Consolidation): 겹치는 지식들을 하나로 융합해 복합형 문제로 업그레이드합니다.

 3단계 (Verification): 앞뒤 맥락 없이는 못 푸는 대명사를 지우고 문장 자체로 완벽하게 다듬습니다.

 4단계 (Entity Surfacing): "A는 B다"를 외우면 "B는 A다"를 못 맞추는 AI 특유의 멍청함(역전의 저주)을 깨기 위해, 속성을 주고 거꾸로 대상을 맞추는 문제를 만듭니다.

 5단계 (Cross-Document Synthesis): 1번 문서의 단서와 5번 문서의 단서를 엮어야만 풀 수 있는 '초고난도 수능형 복합 문제'를 생성합니다.

이렇게 만들어진 66만~164만 개의 연계 문제집을 가지고 소형 모델에게 오직 질문과 답변만 보고 외우도록 SFT 학습을 시킵니다. 덕분에 비서 모델은 지식을 단순히 복사하는 게 아니라 자신의 가중치 파라미터 공간에 개념 구조로 압축하여 내재화하게 됩니다.


3. 숫자로 보는 압도적 성능: RAG 기법의 완벽한 격파

장편 소설 추론의 기적 (NarrativeQA): 긴 맥락 이해가 필요한 이 테스트에서 기존 최고 수준의 그래프 기반 RAG(HippoRAG2)는 23.21%에 그쳤습니다. 문서를 쪼개는 한계 때문이죠. 

반면 MEMO(Gemini-3-Flash + Qwen-14B 메모리) 조합은 53.58%를 기록하며 2배 이상 압도했습니다.

쓰레기 데이터 속에서의 맷집 (노이즈 방어력): 지식 코퍼스에 아무 상관 없는 방해 문서를 섞어 노이즈를 쳤을 때, 기존 RAG 기법들은 6.22%씩 정답률이 뚝뚝 떨어지며 철저하게 낚였습니다. 

반면 MEMO는 지식을 이미 개념으로 암기하고 있어 노이즈가 섞여도 성능 변화가 단 0.55% 내외에 불과했습니다. 완벽한 맷집입니다.

1.2B 체급으로 1.5B를 후려치는 가성비 (MuSiQue 벤치마크): 다중 홉 factual 추론 테스트에서 표준 트랜스포머 구조인 Qwen2.5-1.5B 메모리는 42.90%를 기록했습니다. 

반면 16개 레이어 중 10개를 선형 합성곱 블록으로 채운 하이브리드 아키텍처 모델(LFM2.5-1.2B)을 메모리로 썼을 때는 파라미터(용량)가 20% 더 작음에도 45.23%로 오히려 Qwen을 앞섰습니다. 

국소 팩트를 촘촘하게 압축하는 데 하이브리드 구조가 엄청난 가성비를 낸다는 뜻입니다.


4. 무거운 데이터 업데이트 비용과 뚜렷한 한계점

물론 세상에 공짜 마법은 없습니다. 데이터 관리 관점에서 냉정하게 장단점이 갈립니다.

 - 업데이트의 비효율성: 새로운 문서가 추가될 때 실시간으로 벡터 DB에 밀어 넣으면 끝나는 RAG와 달리, MEMO는 매번 스타 강사 LLM으로 문제집을 새로 합성하고 소형 모델을 다시 SFT 학습(수십~수백 GPU-hours 소요)시켜야 하므로 확연히 무겁고 비효율적입니다.

가중치 머징(Model Merging): 재학습 대신 새 데이터용 모델만 학습시켜 기존 가중치와 결합(Merge-TIES)하면 연산 비용을 33% 아낄 수 있지만, 11~19%의 정답률 하락이 발생합니다.

 - 출처 추적의 불투명성: RAG는 답변과 함께 원문 문서 링크를 사용자에게 직접 띄워줄 수 있지만, MEMO는 지식을 파라미터 공간에 녹여 자연어로 대화하기 때문에 이 답변이 정확히 몇 페이지 어느 문장 유래인지 기원을 명확히 증명하기 어렵습니다.


마치며: 어떤 인프라 선택을 해야 할까?

 - "데이터가 분 단위로 시시각각 바뀌고, 사용자에게 근거 원문 링크를 명확히 띄워줘야 한다"면 기존 RAG가 여전히 정답입니다.

 - "사내 규정집, 의학/법률 매뉴얼처럼 지식 추가 주기가 고정되어 있고, 대신 RAG가 절대 못 푸는 복잡한 복합 문서 추론을 완벽하게 풀어내면서 실시간 상용 LLM API 비용을 극적으로 아끼고 싶다"면 MEMO 방식이 압도적인 해답이 됩니다.

가성비 팁을 드리자면, 4-bit 양자화(GGUF)를 적용할 경우 메인 지휘관인 Qwen-32B(Q4_K_M)는 약 20.2GB, 메모리 비서인 LFM-1.2B(원본 FP16 유지 권장)는 약 2.4GB로 세팅할 수 있습니다.

이 조합은 필수 버퍼를 포함해도 VRAM 30GB대 초반으로 컷당하므로, 비싼 H100 없이 RTX 4090 1장만으로 차세대 대화형 RAG 시스템 구동이 가능해집니다.

화려한 수학적 변형보다 한정된 하드웨어 자원에서 지식을 어떻게 파라미터 공간에 효율적으로 압축하고, 영리하게 인터뷰로 꺼내 쓸 것인가라는 공학적 질문이 만든 결과입니다. 나중에 구현해봐야겠습니다.
