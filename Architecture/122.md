# 만능 옴니모델 vs 도메인 특화 모델: KDL Frontier

우리가 비싼 OCR VLM을 쓰는 이유는 계약서, 다단 논문, 복잡한 표 등 비정형 데이터의 구조화와 손글씨 인식 때문입니다. 단순 타이핑된 PDF에서 글자만 뽑아낼 목적이라면 기존 툴(PyMuPDF 등)이 속도도 빠르고 정확도도 높습니다.

하지만 현재 널리 쓰이는 모델들은 이 지점에서 구조적인 모순을 드러내고 있습니다.

## 1. 범용 옴니모델의 함정: '비대한 자아'와 '텍스트 편향'

현재 최강자로 불리는 제미나이 같은 옴니모델을 써보면 답답할 때가 많습니다. 텍스트와 표는 기가 막히게 읽어내지만, 연산 비용이 높은 차트나 이미지는 슬쩍 무시해버리는 텍스트 편향 현상이 잦습니다.

더 치명적인 것은 과도한 안전 정렬과 자의식 과잉입니다. "표를 절대 요약하지 말고 있는 그대로 추출해"라고 명령해도, 모델 스스로 "요약해 주는 게 사용자에게 더 유익하겠지?"라며 임의로 문서의 실제 값을 훼손합니다. 

문자를 잘 읽고 문서 구조를 잘 파악하고, 사용자 명령을 무시하는 황당한 현상이 가끔씩 발생합니다. 당연히 잘 하는 순간도 많지만, 텍스트 추출 도구가 사용자 명령을 무시하는 것이 말이 되나요?

글자 하나가 중요한 OCR 환경에서 텍스트 추출 도구가 사용자 명령을 무시하고 문맥을 훼손하는 것은 그야말로 최악의 단점입니다.

저는 여러 논문을 제미나이와 같이 보는데, 이럴 때는 좀 열받습니다.

## 2. 특화 모델 KDL Frontier의 압도적 안정성

범용 모델들이 이렇게 헤매고 있을 때, OCRBench v2에서 Google의 차세대 모델을 꺾고 1위를 차지한 모델이 있습니다. 바로 Korea Deep Learning Inc.의 'KDL Frontier'입니다.

 - 평균 점수: KDL Frontier (68.1점) vs Gemini 3 Pro Preview (63.4점)
 - 요소 파싱(표/차트 구조화): KDL (40.7점) vs Gemini (27.1점)
 - 수학적 계산: KDL (73.1점) vs Gemini (49.7점)

가장 인상적인 점은 안정성입니다. 특정 태스크에서 점수가 널뛰는 다른 모델들과 달리, 한국딥러닝의 KDL Frontier는 모든 영역에서 균일하게 최상위권 점수를 유지합니다.

언제나 문제에 맞게 특화된 소형 모델들이, 해당 태스크에서는 최고의 성능을 발휘합니다.

## 3. 8B~12B 오픈 모델들의 반란과 남은 과제

과거 소형 오픈 모델들은 표나 그래프를 만나면 제대로 인식도 못했습니다. 

하지만 2026년 현재, 100B 이상의 무거운 모델 대신 8B~12B 체급의 특화 모델(SLM)들이 아키텍처 혁신으로 60점대 벽을 돌파하고 있습니다.

- Qwen3.5-9B (종합 58.7점): 바운딩 박스와 텍스트를 묶는 '절대 좌표 인코딩' 공간 지각력을 극대화

- GLM-4.6V-Flash (종합 59.0점): 의미 단위로 잘라내는 '동적 해상도 분할' 

- Nemotron Nano V2(종합 61.2점): 시각 토큰의 '고효율 압축' 고해상도 문서를 세밀하게 읽어냄.

이를 통해 공간 지각력을 높이고 고해상도 문서를 세밀하게 읽어냅니다.

아쉽게도, 위치 지정(Referring) 테스트에서 특화 학습한 Qwen3.5-9B(52.1)가 KDL Frontier(60.8)나 Gemini(65.9)에 비해 낮은 성능을 보입니다.

다행인 점은, 수학적 계산(Calculation) 테스트에서 GLM-4.6V-Flash(90.4)나 Nemotron Nano V2 VL(80.4)가 KDL Frontier(73.1)나 Gemini(49.7)보다 나은 성능을 보입니다. 반대로 KDL Frontier나 Gemini가 발전할 여지가 이 부분에 있습니다.

가장 아쉬운 점은, 로컬 환경에서 8B~12B는 OCR로 쓸 수 없을 수준으로 너무 거대하고, 3B 이하의 소형 모델들은 성능이 많이 부족하니... 기묘한 정체기에 머물러 있습니다.

## 4. 앞으로 오픈 모델이 나아갈 미래

결국 오픈 모델이 나아가야 할 방향은 맹목적인 파라미터 확장이 아닌, 태스크에 맞는 아키텍처와 입출력의 변환입니다.

- 입력의 변화 (AwaRes): 문서 전체를 고해상도로 밀어 넣는 낭비를 버려야 합니다. 뼈대만 먼저 훑고, 필요한 곳에만 지능적으로 '돋보기(Crop)'를 들이대어 연산 효율을 극대화해야 합니다.

- 출력의 변화 (MinerU-Diffusion): 순서대로 한 글자씩 뱉어내는 AR 방식에서 벗어나야 합니다. dLLM 모델을 통해 픽셀 정보를 한 번에 병렬 디코딩하면, 빠르고 정직하게 픽셀만 읽어내는 OCR이 가능해집니다.

- 구조의 변화 (CNN + ViT 하이브리드): 대부분의 모델들은 문서 전체의 숲(레이아웃)을 보는 데 능한 ViT 구조에 의존합니다. 그러다 보니 작은 글씨의 정확한 픽셀 위치(나무)를 찾는 데는 시력이 떨어집니다. 반면 CNN은 미세한 픽셀과 윤곽선을 잘 잡아냅니다. 

NVIDIA Nemotron3, Qwen 3.5, Liquid AI LFM2 같은 모델들이 텍스트 생성에서 하이브리드 구조의 장점을 증명했듯, VLM 역시 CNN과 ViT를 결합한 하이브리드 구조로 진화하는 것이 맞습니다.

CNN으로 글자의 픽셀을 또렷하게 읽고, ViT로 문서 전체의 구조를 조망하는 하이브리드 인코더가 경량화와 정밀도를 모두 잡는 뼈대입니다.

## 마치며: 만능 옴니모델 vs 도메인 특화 모델

제미나이 같은 옴니모델은 일상 대화에서 훌륭한 제너럴리스트입니다. 하지만 정확성이 생명인 OCR 태스크에서는, 사용자의 말을 무시하는 덩치 큰 모델 보다 데이터의 본질에 맞게 아키텍처를 깎아낸 KDL Frontier 같은 특화 모델이 더 나은 퍼포먼스를 보여줍니다.

2D 공간의 맥락을 완벽히 이해하는 특화 파이프라인을 구축하는 것. 이것이 2026년 문서 AI의 가장 명확한 이정표입니다. 

오픈 모델들도 단순히 덩치를 키우는 것이 아니라, 이러한 구조적 혁신을 수용할 때 비로소 로컬 OCR의 진정한 대중화가 열릴 것입니다.
