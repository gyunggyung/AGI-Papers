# 터미널 에이전트, 1B(8B A1B) 모델이 뭘 할 수 있을까요?

새벽에 25.53달러를 써서, 터미널 전용 에이전트를 만들었습니다. Liquid AI의 LFM2-8B-A1B 모델을 NVIDIA의 Nemotron-Terminal-Corpus 데이터셋으로 VESSL AI에서 Unsloth AI를 써서 학습했습니다. 

아직 SFT 과정의 1/4도 못 했는데 성과가 괜찮게 나와서 미리 공유합니다.

### 🎯 목표: 로컬 LLM Claude Code

제 노트북 윈도우 CPU(RAM 16GB) 환경에서도 동작하는, Claude Code처럼 터미널 명령어를 써서 시스템을 조작하는 터미널 에이전트를 구현해보려고 했습니다.

### 🚀 실전 테스트 결과 (Step 450)

어떻게 동작하는지 3가지 예시를 보여드리겠습니다.

1. **패키지 설치 및 서버 운영 (Score 4.5/5.0)**
- 미션: "nginx를 설치하고 서비스로 등록한 뒤, 실제로 돌고 있는지 확인해줘."
- 모델의 반응: apt(설치) -> systemctl(활성화) -> netstat(포트 확인)
- 감상: 단순히 하나만 하는 게 아니라, 확인까지 마치는 작업의 완결성을 보여줍니다.

2. **보안 및 파일 검증 (Score 4.5/5.0)**
- 미션: "특정 URL에서 파일을 받고, 체크섬(SHA256)을 확인한 뒤 압축을 풀어줘."
- 모델의 반응: mkdir -> cd -> wget -> sha256sum -> tar
- 감상: 경로를 미리 만들고 이동해서 작업을 수행하는 정석적인 흐름을 보여줍니다.

3. **방화벽 설정 및 포트 개방 (Score 4.5/5.0)**
- 미션: "현재 서버의 방화벽 상태를 점검하고, HTTPS 통신을 위해 443번 포트를 열어줘."
- 모델의 반응: ufw status(점검) -> ufw allow 443(포트 개방)
- 감상: 명령어 하나만 틀려도 서비스가 중단될 수 있는 보안 설정에서도 정확한 도구(ufw, iptables)를 찾아내어 안전하게 실행합니다.

### 🧠 지능의 징후: "일단 좀 볼게요 (Exploration)"

가장 흥미로운 부분은 모델의 신중함입니다. 데이터를 처리하라는 명령을 받으면 바로 복잡한 파이프라인(cut, awk 등)을 쓰지 않습니다. 

대신 'ls'와 'head'를 먼저 입력해 파일이 정말 있는지, 데이터가 어떻게 생겼는지 확인부터 합니다. 훈련 데이터에서 이런 지능적인 탐색 패턴이 관찰되고 있습니다.
