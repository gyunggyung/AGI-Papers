# Vibe Coding (바이브 코딩)

> **"코드는 잊어라. 무드(Vibe)를 관리해라." Andrej Karpathy가 던진 프로그래밍의 새로운 화두.**

## 1. Vibe Coding이란?

2025년 2월, Andrej Karpathy는 새로운 코딩 패러다임을 제안합니다.
*   **정의**: 개발자가 코드를 한 줄 한 줄 짜는(Writing) 것이 아니라, AI에게 자연어로 "이런 느낌으로 만들어줘"라고 지시하고, 결과물을 눈으로 확인하며 수정해 나가는 방식.
*   **핵심**: 코드는 **부차적인 것(Implementation Details)**이 됩니다. 중요한 것은 **실행 결과(Output)와 사용자의 의도(Intent)**입니다. 복잡한 로직은 AI가 알아서 하고, 인간은 "이거 되네? 아니, 좀 이상한데?"라는 **직관(Vibe)**으로 개발을 주도합니다.

---

## 2. 빛과 그림자

### The Bright Side (생산성의 폭발)
*   **속도**: 숙련된 개발자는 타자를 칠 필요가 없어집니다. 말하는 속도로 앱이 만들어집니다.
*   **접근성**: 코딩을 모르는 일반인도 자신의 아이디어를 소프트웨어로 구현할 수 있습니다.
*   **Focus**: `for` 문을 어떻게 돌릴지 고민하는 대신, "이 앱이 사용자에게 어떤 가치를 줄까?"에 집중할 수 있습니다.

### The Dark Side (영구적인 주니어의 함정)
하지만 Karpathy는 경고합니다.
*   **Eternal Junior**: AI가 짜준 코드를 이해하지 못하고 "그냥 되니까 쓴다"는 태도로 일관하면, 영원히 초보 수준에 머물게 됩니다.
*   **Black Box**: 버그가 발생했을 때, 코드를 읽을 줄 모르면 아무것도 할 수 없습니다. "AI님, 고쳐주세요"라고 빌어야 합니다.
*   **Security**: AI가 보안 구멍(API Key 노출, SQL Injection 등)을 만들어도, 'Vibe'만 좋으면 그냥 배포해버릴 위험이 있습니다.

---

## 3. 결론: 매니저가 되어라

Vibe Coding은 거스를 수 없는 흐름입니다. 하지만 이것이 **"공부를 안 해도 된다"는 뜻은 아닙니다.**
오히려 반대입니다. AI가 쏟아내는 코드의 홍수 속에서 옥석을 가려내고, 보안/성능/아키텍처를 검수(Review)하려면 **더 탄탄한 기본기(Fundamentals)**가 필요합니다.

이제 개발자는 '코더(Coder)'에서 **'AI 테크 리드(AI Tech Lead)'**로 진화해야 합니다. 직접 벽돌을 쌓는 사람이 아니라, 벽돌 쌓는 로봇들을 지휘하는 감독관이 되어야 합니다.
