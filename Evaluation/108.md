Step-3.5, LLaDA2.1, Qwen3.5-397B, ZAYA1-74B, Titan-14B... 최근 몇 주간 거의 86개에 달하는 최신 모델들을 로컬 컴퓨터와 테스트 서버에 올려서 직접 굴려보았습니다. 

이 무식하고 피곤한 짓을 하며 뼈저리게 느낀 4가지 솔직한 점들을 공유합니다.

1. 오픈소스 벤치마크는 완전히 오염되었습니다. (사기극의 종말)

허깅페이스 점수나 평가 시트(AIME, GSM8K 등)에서 90점을 넘었다고 자랑하는 모델들을 로컬에 얹어 실행해 보면, 정작 간단한 한국어 대화나 아주 기초적인 리눅스 쉘 명령어 매칭조차 제대로 못 하고 버벅거리는 경우가 허다합니다. 

데이터셋 누출(Data Leakage)을 통해 벤치마크 문제집을 통째로 외운 '기출문제 암기형' 가짜 똑똑이 모델들이 판을 치고 있습니다. 점수판의 순위는 이제 마케팅용 껍데기일 뿐입니다.

2. 한국어 데이터 파이프라인의 치명적 부재

외산 베이스 모델에 한국어 SFT를 살짝 얹어 낸 모델들 대부분이 '한국어의 탈을 쓴 번역기' 수준을 못 벗어납니다. 문법은 맞지만 문맥이 어색하고, 특히 리얼타임 코딩이나 시스템 조작 같은 실무 환경에 던져두면 극도로 취약해집니다. 

데이터의 양도 문제지만, 한국어 언어 모델만을 위한 고순도 인스트럭션(Instruction) 데이터 파이프라인의 정밀한 설계가 너무나도 절실합니다.

3. 결국 본질은 베이스 모델의 지능 체급

SFT(지도 미세조정)나 LoRA 튜닝은 모델에게 똑똑함을 가르치는 과정이 아닙니다. 베이스 모델이 이미 가지고 있는 지식 중에서 \"이런 형식으로 이쁘게 대답해 줘\"라고 포맷팅(포장)을 가르치는 것에 가깝습니다. 

베이스 모델 자체의 지능(Pre-training 체급)이 떨어지면, 아무리 좋은 데이터로 SFT를 100번 돌려도 결국 실무에서는 밑천이 드러납니다. 튜닝의 기교보다 좋은 베이스 모델을 고르는 선구안이 훨씬 중요합니다.

4. 그럼에도 Qwen3.5와 LFM이 보여준 희망

이 난장판 속에서도 빛나는 진짜들이 있었습니다.
- Qwen3.5 계열: 베이스 모델 자체의 코딩 및 JSON 출력 지능이 압도적입니다. 대충 튜닝해도 실무 에이전트 역할을 훌륭히 수행합니다.
- LFM(Liquid AI) 계열: 기존 트랜스포머가 아닌 다른 아키텍처임에도, 튜닝에 따른 지능 학습 효율이 매우 뛰어납니다. 적은 비용으로 놀라운 퍼포먼스를 냅니다.


마치며: 기교를 버리고 본질로

모두가 하루가 다르게 새로운 모델을 출시하며 SOTA(최고 성능)를 달성했다고 외치지만, 현장에서 직접 검증해 보면 실망감이 가득합니다. 

이런 시대일수록 점수판의 숫자에 휘둘리지 않는 냉정한 검증 능력이 필요합니다. 껍데기뿐인 튜닝 기교에 시간과 리소스를 낭비하지 않고, 진짜 지능을 가진 베이스 모델을 선별하고 그에 맞는 고순도 실무 데이터를 깎아내는 것만이 실무에서 가치를 만드는 유일한 길입니다.
