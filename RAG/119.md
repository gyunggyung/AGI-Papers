title: "SLIDERS: LLM의 환각과 컨텍스트 한계를 제어하는 SQL 데이터베이스 구조"
---
스탠포드 연구진이 발표한 SLIDERS 논문은 LLM의 환각과 컨텍스트 길이의 한계를 SQL 데이터베이스 구조로 통제합니다.

기존의 RAG나 롱 컨텍스트 모델은 도서관 사서(LLM)에게 수만 페이지의 책을 주며 "전부 머릿속에 외워서 요약해!"라고 강요하는 것과 같습니다.

정보가 섞이고 누락되는 집계 병목 현상이 필연적으로 발생합니다.

반면 SLIDERS는 사서에게 엑셀을 쥐여줍니다.

텍스트를 읽고 '이름', '연도', '내용' 칸에 데이터를 표로 정리(RDB)하게 한 뒤, 질문이 오면 머리를 쥐어짜는 대신 명확한 SQL 필터를 돌려 답을 찾게 합니다.

1. 압도적인 스케일과 비용의 마법 (3,600만 토큰의 벽)

동적으로 텍스트를 탐색하는 RLM 같은 재귀적 에이전트는 연산이 폭발해 3,600만 토큰 벤치마크(FinQ100)에서 구동을 포기해야 했습니다. (예상 비용 2,000달러).

하지만 SLIDERS는 텍스트를 구조화된 표로 고정시켜 버리기 때문에, 단 34달러로 55.22%라는 최고 수준의 정확도를 달성했습니다.

2. 오픈소스의 퀀텀 점프

오픈 모델(Qwen 122B)에 이 파이프라인을 적용하자, 평균 정확도가 61.4%에서 75.2%로 무려 14%p 높아졌습니다.

거대 API에 의존하지 않고도, 구조만 잘 짜주면 오픈소스 생태계에서도 충분히 승산이 있음을 증명했습니다.

3. 속도의 재정의: 지루하지만 가장 확실한 길

대중이 열광하는 엔터를 치면 1초 만에 나오는 마법은 아닙니다.

초거대 문서 100개를 처음 읽고 표로 전처리하는 약 16분이 걸립니다. 모호한 자연어를 SQL 규격으로 깎는 것은 고통스럽고 지루한 작업입니다.

하지만 이 과정을 한 번만 참아내면, 이후 쏟아지는 복잡한 질문들은 단 25초 만에 정확하게 쿼리해 냅니다.

결론적으로, 이 논문은 AI를 대충 그럴싸한 답을 내는 마법 지팡이로 포장하지 않습니다.

대신 인간이 언제든 논리의 뼈대를 추적하고 출처를 검증할 수 있는 투명한 톱니바퀴를 만들었습니다. 

제한적인 로컬 환경에서 우리는 무한 컴퓨팅을 숭배할 필요가 없습니다.

데이터를 투명하게 정제하고, 룰을 세우고, 가벼운 에이전트가 그 위에서 뛰놀게 하면 됩니다. 낡은 줄 알았던 SQL의 시대가 LLM의 손을 잡고 가장 세련된 형태로 돌아오고 있습니다.
