# Text-to-LoRA & Doc-to-LoRA: 즉각적 모델 업데이트

> **"문서 내재화에 0.5초, 새로운 스킬 장착에 1초 미만."** Sakana AI가 Text-to-LoRA와 Doc-to-LoRA를 통해 모델 업데이트의 패러다임을 바꿨습니다.

LLM에게 새로운 지식을 가르치거나 특정 작업을 지시할 때, 우리는 늘 두 가지 거대한 한계에 부딪힙니다. VRAM 폭발과 지연 시간, 그리고 값비싼 GPU 자원과 데이터 수집 시간.

사카나 AI(Sakana AI)는 이 문제를 '비용 상각(Cost Amortization)'이라는 철학으로 돌파했습니다. 무겁고 비싼 최적화(학습) 과정을 단 한 번의 가벼운 '정방향 연산(Forward pass)'으로 대체한 것입니다.

## 1. 스킬의 즉각적 장착 (Text-to-LoRA)

*   **작동 방식**: "이 작업은 수학적 추론을 통해 문제를 해결하는 것입니다"라는 짧은 자연어 지시문(Task description)만 입력합니다.
*   **결과**: 하이퍼네트워크(어댑터 생성기)가 이 문장을 읽고, 타겟 모델을 해당 작업에 맞게 조향하는 LoRA 가중치를 **1초 미만**의 시간에 즉시 생성해 냅니다.
*   **의의**: 번거로운 데이터 큐레이션 없이, 프롬프트 한 줄 작성하는 것만으로 수백 개의 작업 특화 모델을 온디맨드로 찍어낼 수 있습니다.

## 2. 지식의 즉각적 내재화 (Doc-to-LoRA)

*   **작동 방식**: 수만 토큰에 달하는 긴 문서(심지어 VLM이 추출한 시각 정보까지)를 하이퍼네트워크에 통째로 밀어 넣습니다.
*   **결과**: **0.5초 이내**에 문서의 사실적 지식이 압축된 LoRA 어댑터가 구워집니다.
*   **의의**: 일단 어댑터가 씌워지면, 다음 질문부터는 원본 문서가 프롬프트에 없어도 완벽하게 답변합니다. 매 쿼리마다 발생하던 막대한 KV 캐시 메모리 소모가 사라지며, 타겟 모델의 기본 문맥 길이를 4배 이상 초과하는 정보도 장기 기억으로 보존합니다.

## 3. 공장 짓기와 물건 찍어내기 (하드웨어의 현실)

"그 엄청난 압축이 어떻게 1초 만에 가능한가?" 비밀은 역할 분담에 있습니다.

*   **공장 짓기 (메타 학습)**: 어떤 문서나 텍스트가 들어와도 완벽한 LoRA를 뱉어내는 이 '마법의 생성기'를 최초로 짓는 데는 H200 GPU 8대로 5일(D2L 기준)이 걸렸습니다.
*   **물건 찍어내기 (추론)**: 하지만 한 번 완성된 생성기의 파라미터는 5M~309M 수준으로 매우 작습니다. 실서비스 배포 시에는 거창한 GPU 학습 없이, **CPU 환경에서도** 가벼운 순전파 연산 한 번으로 즉석에서 어댑터를 만들어냅니다.

## 4. 로컬 에이전트의 구원투수 (Sleep-time Compute)

이 논문들이 진정으로 빛나는 곳은 리소스가 극도로 제한된 온디바이스 로컬 환경입니다.

*   **낮 (활성 상태)**: 에이전트가 업무를 수행할 때 Text-to-LoRA를 호출해, 현재 작업 성격에 맞춰 순식간에 모델의 스킬을 교체하며 일합니다.
*   **밤 (유휴 상태)**: 업무가 끝나고 모델이 쉴 때, 하루 동안 쌓인 문서나 대화 로그를 Doc-to-LoRA로 압축하여 가중치 형태의 영구 기억으로 저장합니다.

이른바 '수면 시간 연산' 아키텍처입니다. 시간당 수십 달러씩 태우는 거대 API에 의존하지 않고도, 내 노트북 안에서 매일 지식을 축적하고 개인화되는 AI를 운영할 수 있게 됩니다.

## 💡 마치며: 패러다임의 전환

지금까지 우리는 모델의 덩치를 무작정 키우거나(Dense), 전문가를 나누는(MoE) 구조적인 확장에만 몰두해 왔습니다. 하지만 사카나 AI는 "훈련(Training) 자체를 생성(Generation)으로 대체한다"는 완전히 새로운 모듈식 업데이트 API의 미래를 열었습니다.
