---

사람은 수면을 통해 낮에 본 단기 기억을 대뇌피질(장기 기억)로 옮기고 논리망을 단단하게 구축합니다. 그런데 LLM도 똑같다면 어떨까요? 카네기 멜런(CMU)과 메릴랜드 대학교 연구진이 발표한 "Language Models Need Sleep"은 언어 모델에게 생물학적 수면 메커니즘을 적용해 LLM의 추론 능력을 폭발적으로 끌어올렸습니다.

1. 메모리 용량이 아니라 생각할 시간이 문제다

요즘 LLM은 수만 자의 글을 한 번에 읽습니다. 하지만 트랜스포머의 일시적 기억 공간(KV 캐시)이 꽉 차면 옛날 내용을 지워버려야 합니다. 이를 해결하려 고정된 압축 메모리를 쓰는 하이브리드 모델(SSM)들이 나왔지만, 여전히 수학이나 복잡한 논리 문제는 젬병이었습니다.

논문은 지적합니다. "기억할 방이 좁아서가 아니라, 들어온 정보를 씹어 삼킬 시간(연산량)이 부족했던 것이다."

2. 트랜스포머가 읽고, SSM이 잠자며 학습한다

이 문제를 풀기 위해 논문은 트랜스포머(Attention)와 SSM이 번갈아 섞인 하이브리드 아키텍처를 활용합니다. 두 레이어의 역할 분담은 명확합니다.

 - 트랜스포머 레이어: 사용자가 입력한 최신 컨텍스트를 고화질로 읽고 추론하는 눈의 역할을 합니다.


연구진은 이 검증을 위해 엔비디아의 하이브리드 모델인 'Jet-Nemotron 2B'와, 순수 트랜스포머 모델에 SSM 레이어 6개를 이식해 개조한 'Ouro 1.4B' 모델을 가져와 직접 수면 파인튜닝 실험을 진행했습니다.

3. AI의 수면(Sleep) 메커니즘: 멈추고, 반복하고, 새겨라


 - 외부 단절: 새로운 텍스트를 읽는 것을 완전히 멈춥니다.

 - 심층 복습: 방금까지 트랜스포머가 읽은 문맥을 가지고 내부적으로 2번~6번씩 반복 연산(루프)을 돕니다.

 - 지식 각인: 단기 기억에 있던 파편화된 정보를 모델의 단단한 뼈대인 빠른 가중치(Fast Weights)에 깊게 박아 넣습니다.

 - 청소 후 기상: 뇌에 완벽히 새겼으니, 임시 캐시를 깨끗이 비우고 홀가분하게 다음 문단을 읽습니다.

4. 답변 속도는 그대로인데 지능은 52% 껑충 (숫자로 보는 성과)

사용자가 체감하는 답변 속도는 어떨까요? 기존의 엄청나게 빠른 속도가 100% 그대로 유지됩니다. 미리 잠자는 동안 복잡한 추론을 다 끝내놨기 때문에, 질문을 받으면 단 한 번의 가벼운 계산(Single forward pass)만으로 천재적인 정답을 뱉어냅니다.

 - 초고난도 수학 (GSM-Infinite): 14억 파라미터의 소형 상용 모델에 4번의 수면을 주었더니, 정답률이 41.9%에서 61.5%로 수직 상승했습니다. (무려 47% 향상)

 - 제약된 슬라이딩 윈도우 환경: 메모리 창이 좁은 극한의 상황에서는 59.6%였던 정답률이 90.5%로 폭등했습니다. (+52%) 밀려나서 지워질 뻔한 단서들을 잠자는 동안 꾹꾹 압축해 낸 결과입니다.


5. 세상에 공짜는 없다: 치명적인 딜레마와 인프라 비용


무엇보다 학습(파인튜닝) 과정이 끔찍합니다. 남들 한 번 읽고 지나갈 때 N번씩 역전파 연산을 해야 하니 H100 같은 초고가 GPU가 배로 필요하고, 훈련 중 그라디언트가 터져 에러가 날 확률도 급증합니다. 연구진도 Muon 옵티마이저와 메모리 체크포인팅 기법을 영혼까지 끌어모아 겨우 훈련에 성공했습니다.


마치며: 에이전트 꼼수에서 근본 아키텍처로의 회귀

그동안 우리는 AI가 복잡한 생각을 하게 만들려고 "생각 중..." 이라며 텍스트를 길게 뽑아내는 Chain-of-Thought에 의존했습니다. 비용도 비싸고 속도도 느렸죠.

하지만 이 연구는 "추론에 필요한 연산을 겉으로 보이는 출력 단계가 아니라, 뇌(가중치) 내부의 입력 단계로 밀어 넣는" 근본적인 모델링의 혁신을 보여줬습니다.

당장 실무 서비스에 완벽히 적용하기엔 무겁고 까다롭지만, "LLM에게도 깊은 생각을 위한 본질적인 순차 연산(수면) 시간이 필요하다"는 것을 구조적으로 증명한 아주 우아하고 묵직한 연구입니다. 올해 말에는 어떤 연구와 어떤 모델이 나올지 기대됩니다.
