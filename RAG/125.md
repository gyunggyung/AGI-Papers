title: "MRAgent: Memory is Reconstructed, Not Retrieved"
---

인간의 뇌는 과거 기억을 컴퓨터 파일 열듯이 수동적으로 인출(Retrieve)하지 않습니다. 단서를 디딤돌 삼아 기억의 꼬리를 물며 능동적으로 재구성(Reconstruct)하죠. 

벡터 DB에 기반한 단순한 RAG는 질문과 단순히 겉보기만 비슷한 텍스트 파편들을 무작정 긁어와 프롬프트에 집어넣다 보니, 쓸데없는 노이즈만 가득 차고 정작 중요한 맥락은 놓치기 일쑤입니다.

NUS 연구진이 ICML 2026에서 발표한 "Memory is Reconstructed, Not Retrieved: Graph Memory for LLM Agents"은 인간의 뇌와 같은 인지과학적 지혜를 LLM 에이전트의 외부 메모리 시스템에 완벽하게 이식했습니다. 'MRAgent'의 핵심을 쉽게 풀어드립니다.

1. 생각하며 기억의 지도를 찾아가는 Agentic GraphRAG

MRAgent는 대화나 문서 기록을 단순한 텍스트 덩어리로 쌓아두지 않습니다. 대신, 인간의 기억 신경망을 닮은 이종 그래프(Heterogeneous Graph)로 구조화합니다.

 - 단서 (Cues): 인물, 장소, 핵심 키워드

 - 연상 태그 (Tags): 단서와 실제 구체적 기억 내용을 연결하는 이름표이자 의미적 다리

 - 내용 (Content): 타임라인 선상의 에피소드(사건), 취향이나 성격 같은 안정적인 의미 지식, 그리고 상위 주제별 추상화 레이어로 쪼개진 실제 기억 정보

가장 큰 차이는 정답을 찾아가는 과정입니다. 기존 RAG가 질문을 던지면 관련 문서들을 한 번에 와르르 가져오고 끝나는 수동적 방식이라면, MRAgent는 LLM이 탐색용 전용 툴킷을 쥐고 그래프 위를 직접 징검다리 건너는 능동적 방식입니다.

"네이트가 세 번째 토너먼트에서 우승했을 때 상금을 어디에 썼어?"라는 질문이 들어오면, MRAgent는 한 번에 정답을 뒤지는 것이 아니라, '네이트', '우승' 노드에서 출발해 연상 태그를 타고 관련 에피소드들을 모읍니다. 

그 후 "잠깐, 어떤 게 세 번째 사건이지?"라고 스스로 판단하여 대화 시간 확인 도구를 호출해 순서를 검증하고, 그 뒤에 숨겨진 상금 저축 내역을 정확히 역추적해 냅니다. 즉, 추론을 하면서 동시에 다음 검색 단계를 결정하는 루프를 돌리는 것입니다.

2. 압도적인 성능 숫자의 격차: 최대 23%의 향상

이론만 그럴싸한 게 아닙니다. 길고 복잡한 대화 기록을 이해해야 하는 LoCoMo 벤치마크와 수십 개의 세션이 누적된 대용량 LongMemEval 벤치마크에서 기존 최고 수준의 메모리 모델들을 가볍게 눌렀습니다.

 - 종합 정확도 대폭 상승: Gemini-2.5-Flash 백본 기준, 기존 최고 메모리 모델(Memo, 68.31%) 대비 절대치로 15.9%p 상승한 84.21%를 기록했습니다. 상대적 성능 향상 폭은 무려 23.3%에 달합니다.

 - 강력한 모델에서도 검증: 훨씬 더 똑똑한 Claude-Sonnet-4.5 백본 위에서도 기존 최고 에이전트 메모리 툴(LangMem, 78.61%)을 제치고 88.32%라는 압도적 점수를 보여주었습니다.

 - 시간 흐름과 다중 추론의 강자: 특히 여러 기억을 징검다리처럼 엮어야 하는 다중 홉(Multi-hop) 질문과, 시간 순서를 꼬아서 물어보는 정합성 테스트에서 기존 모델들을 절대치 기준 6%~18%p 이상 격차로 따돌렸습니다.

3. 연산 시점의 혁신이 만든 가성비: 토큰 소모량 27분의 1로 폭락

기존의 GraphRAG 및 메모리 시스템들은 처음 그래프를 빌드할 때, 돈과 GPU 연산이 엄청나게 많이 듭니다. 문서나 대화가 새로 들어올 때마다 기존의 모든 기록을 다 헤집으며 정밀하게 관계를 추출하고 엣지를 촘촘하게 연결해야 했기 때문입니다.

MRAgent는 이 무거운 연산의 시점을 완전히 바꾸는 공학적 묘수를 썼습니다.

 - 저장할 때는 가볍게: 새로운 대화가 들어오면 무거운 관계 연산 없이, 가볍게 핵심 키워드(Cue)와 이름표(Tag)만 붙여서 정적으로 저장해 둡니다.

 - 검색할 때는 알뜰하게(On-Demand): 질문이 들어오는 시점에만, LLM이 관련 있는 연상 태그(Tag)만 먼저 빠르게 훑어보고 가망이 없는 그래프 줄기 전체를 과감하게 잘라버리는 가지치기(Pruning)를 수행합니다. 본문 전체를 무식하게 읽지 않는 것이죠.

장기 상호작용 평가에서 샘플당 평균 토큰 소모량을 비교했을 때, 기존 시스템인 LangMem이 3,268k 토큰을 쓸 때 MRAgent는 고작 118k 토큰만 소비했습니다. 기억의 정확도는 20% 이상 끌어올리면서 비용은 무려 27배나 아낀 셈입니다.

4. 영리한 설계가 마주한 현실적인 한계와 미래

논문의 마지막 장에서 저자들이 솔직하게 고백한 한계점 역시 우리가 눈여겨봐야 할 공학적 포인트입니다.

관계 추론 연산을 검색 단계로 미루어 비용을 아낀 대가로, 질문이 극도로 복잡해져서 그래프를 아주 깊게 파고들어야 할 때는 여러 번의 도구 호출 루프를 거쳐야 하므로 응답 속도가 다소 늘어난다는 단점이 있습니다. 

또한, 과거 기억을 시간의 흐름에 따라 압축하거나 잊어버리는 망각 메커니즘이 아직 없어서, 대화가 수만 번 누적되면 그래프 크기가 계속 커져 저장 장치 오버헤드가 발생할 수 있습니다.

그래서 저자들은 앞으로 그래프가 무한히 커지지 않도록 영리하게 기억을 통합하는 경량화된 유지보수 기술과 더 견고한 그래프 탐색 알고리즘을 미래 연구 과제로 제시합니다.


마치며: 왜 이 연구에 주목해야 하는가?

수학적 정리(Theorem 4.1)를 통해서도 증명되었듯이, 질문만 보고 한 번에 고정된 데이터를 긁어오는 수동적 방식은, 중간에 찾아낸 단서를 보고 다음 탐색처를 정하는 능동적 방식을 이론적으로 결코 이길 수 없습니다.

메모리를 단서와 태그로 알뜰하게 나눠 저장하고(공간 최적화), LLM이 생각해가며 요구에 따라 영리하게 재구성하는(시간/비용 최적화) 아키텍처적 기본기가 혁신을 만든 것입니다.

이제 2026년입니다. 정적인 데이터 매칭의 시대를 넘어, 스스로 기억을 유추하고 검증하는 생각하는 Agentic RAG의 시대가 열리고 있습니다.
