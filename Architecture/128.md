1,500달러로 7B를 압도한 1B 모델, HRM-Text: 안드레 카파시(Andrej Karpathy)는 고도로 정제된 데이터셋을 통해 인간의 지능적 핵심을 1B 크기의 인지적 코어(Cognitive Core)에 압축해 넣을 수 있으며, 이것이 미래 AGI 시스템의 진정한 두뇌가 될 것이라 말했습니다.

Sapient Intelligence의 HRM-Text 논문은 이 카파시의 혜안을 입증했습니다. 단돈 1,500달러(16 GPU, 1.9일 학습)의 예산과 40B라는 극소량의 토큰만으로 바닥부터 학습된 HRM-Text 1B 모델은 수학과 논리 추론 영역에서 자기 몸집의 수 배가 넘는 모델들을 압도했습니다.

이 가성비와 효율성이 어떻게 가능했는지, 그리고 제가 이 구조를 가져와 한국어 온디바이스 에이전트 환경에 맞게 처음부터 다시 사전학습(Scratch Pretraining)을 진행하고 있는 이유를 공유합니다.

1. 아키텍처 혁신: VRAM은 1인분, 연산 깊이는 8인분

기존 모델들은 똑똑해지려면 가중치가 서로 다른 레이어를 128층씩 일렬로 길게 쌓아야 했습니다. 데이터가 1층부터 128층까지 단 한 번만 직진하고 끝나는 직렬 공장 라인 구조라 GPU 메모리(VRAM)가 폭발합니다.

HRM-Text는 인간의 뇌(전두두정엽 루프)에서 영감을 받아 컴퓨터 메모리에 딱 16층짜리 방 2개(H-모듈, L-모듈)만 선언해 둡니다. 가중치를 고정해 두고, 데이터가 들어오면 이 두 방 사이를 뱅글뱅글 돌리며 재사용(Weight Sharing)합니다.

 - 부장님(H-모듈): 전반적인 주제와 목적지 같은 거시적인 맥락 유지.
 - 대리님(L-모듈): 단어들의 관계나 세부적인 산술 연산 등 국소적인 반복 정제.

질문이 들어오면 두 사람은 정답을 뱉기 전까지 내부적으로 총 8번의 릴레이 회의(H2L3 스케줄)를 나눕니다. 

물리적 파라미터는 딱 1B 수준으로 아주 가볍지만, 토큰이 거치는 실질적인 연산 깊이는 128층짜리 거대 직렬 모델을 통과하는 것과 수학적으로 동일해집니다.

2. 학습법 혁신: 쓸데없는 시험공부는 생략한다

인터넷 잡글을 읽는 사전학습 단계를 아예 생략하고, 처음부터 질문-답변 쌍으로 이루어진 고품질 기출문제집만 가지고 곧바로 사전학습을 시작하는 싱글 스테이지 지시문 사전학습을 도입했습니다.

 - 응답 전용 손실(Response-only Loss): 질문 영역은 시험을 보지 않고, 오직 정답 영역의 토큰들에 대해서만 점수를 매겨 가중치를 고칩니다. 가중치 업데이트 기회를 정답을 만들어내는 힘에만 집중시킵니다.

 - PrefixLM 어텐션 마스크: 질문 영역만큼은 지문 전체를 띄워놓고 보듯 사방으로 눈을 열어 전역 문맥을 양방향(Bidirectional)으로 파악하게 만듭니다.

 - 생각 흔적 강제 제거: 데이터셋에 포함되어 있던 명시적인 생각 흔적(CoT)들을 학습 전에 전부 잘라내 버렸습니다. 

겉으로 중언부언 흉내 내는 대신, 8번의 내부 회의(잠재 공간 연산) 시스템이 뇌 안에서 스스로 깊게 사고하도록 강제하여 강력한 내적 추론 가중치를 형성합니다.

3. 압도적인 성능 수치와 과학적 근거

40B 고품질 토큰만 먹은 HRM-Text 1B의 성적표는 흥미롭습니다. MATH(56.2%), DROP(82.2%), GSM8K(84.5%) 등 고난도 추론 지표에서 자기 몸집의 3~7배인 Llama 3.2 3B, Qwen 3.5 2B, Olmo 3 7B 등을 가볍게 압도했습니다.

Logit-Lens 분석 결과, 마지막 깊은 레이어에 도달할 때까지 숨겨진 상태가 계속해서 유의미하게 변화하며 능동적인 연산을 수행하고 있음이 확인되었습니다.

4. 뼈아픈 한계점: 지식 백과의 부재와 추론 속도

물론 세상에 공짜는 없습니다. 40B라는 적은 데이터셋 스케일 때문에, 머리는 대단히 똑똑하지만 전 세계 만물에 대한 방대한 상식적 지식을 암기하는 능력(MMLU 60.7%)은 조 단위 토큰을 삼킨 모델들에 비해 태생적으로 얕습니다. 똑똑한 계산기일 뿐 걸어 다니는 백과사전은 아닌 셈입니다.

또한 토큰을 생성할 때마다 내부 루프를 8번씩 돌려야 하므로, 가볍기 때문에 VRAM은 적게 차지하지만 실제 연산 부하는 3B~4B급 Dense 모델을 돌리는 것과 맞먹어 추론 속도가 일반 1B 모델보다 느려집니다.

특히 하드웨어 가속이 없는 CPU 온디바이스 환경에서는 이 병목이 답답하게 다가올 수 있습니다.


그래서 저는 이 아키텍처를 가져와 새로 KoHRM-Text-1.4B를 학습하고 있습니다

국내 지식과 법률을 빠삭하게 이해하고, 터미널 셸 커맨드와 JSON 툴콜을 완벽하게 수행하는 실무형 LLM-OS의 두뇌를 완성하려 합니다. 다시 사전학습을 시작했으며, H200 8장으로 학습을 잘 진행하고 있습니다.

논문과의 차이점들입니다:

 - 131K 특화 토크나이저: 한국어 및 JSON 규격이 깨지는 낭비를 막기 위해 토크나이저를 새로 구워 압축률(한국어 2.60자, JSON 3.32자)을 극대화했습니다. 기존 가중치를 버리고 바닥부터 새로 시작한 이유입니다.

 - 법률 및 에이전트 데이터 패킹: 상식 지식의 부재는 한국어 법령/행정규칙/판례 및 위키 데이터 그리고 ToolBench, Nemotron-Terminal-Corpus, 다양한 코딩 에이전트 데이터를 사전학습에 통째로 패킹해 해결합니다.

 - 종착지는 툴콜 검색(RAG)과 ACT의 결합: 모델의 지식이 부족하다면 터미널과 툴콜을 쥐여주면 됩니다. 스스로 검색해 지식의 한계를 씻어내고, ACT(가변 연산)를 결합해 쉬운 문장은 1B 속도로 넘기고 어려운 법리 해석이나 복잡한 에이전트 작업에만 128층의 뇌를 풀가동하는 가변형 에이전트 시스템을 구축하고 있습니다.
