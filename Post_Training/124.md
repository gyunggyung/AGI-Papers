# 그 OpenClaw로 강화학습을? 프린스턴 대학교의 OpenClaw-RL

우리가 ChatGPT나 Claude Code 같은 코딩 에이전트를 쓰다가 "그 파일 말고 다른 거 찾아봐", "에러 났잖아 다시 해"라고 피드백을 주면, AI는 "죄송합니다" 하고 수정된 답을 내놓습니다. 하지만 대화가 끝나면 그 귀중한 피드백(오답 노트)은 시스템에서 그냥 허공으로 날아가 버립니다.

프린스턴 대학교 연구진은 이 '버려지는 신호'를 실시간 학습 데이터로 재활용하는 OpenClaw-RL 프레임워크를 발표했습니다. 우리가 익히 아는 그 OpenClaw를 활용해 AI가 어떻게 스스로 진화하는지, 4가지 핵심과 현실적인 한계를 정리했습니다.

1. 정답지가 없어도 눈치껏 배운다 (Binary RL)

코딩이나 수학과 달리, "AI 티 안 나게 글 써줘" 혹은 "친절하게 채점해 줘" 같은 요청에는 절대적인 정답(Ground Truth)이 없습니다.

이 시스템은 정답지 대신 사용자의 '리액션'을 봅니다. 사용자가 만족하고 다음으로 넘어가는지, 아니면 불만을 표출하며 다시 질문하는지를 평가 모델(PRM)이 채점(+1, -1)하여 실시간 보상을 줍니다. 

명확한 정답이 없는 주관적인 영역에서도 상호작용만으로 학습이 가능해진 것입니다.

2. 거대 교사 모델은 없다: 1인 2역의 마법 (Self-Distillation)

이 논문의 백미는 오류를 구체적으로 고치는 증류(Distillation) 방식입니다. 수천억 파라미터의 거대 외부 모델을 선생님으로 부르지 않습니다. 단일 소형 모델(예: Qwen 4B) 하나가 프롬프트만 바꿔가며 1인 2역을 혼자 다 해냅니다.

 - 생성자 모드 (아무것도 모르는 나): 평소처럼 사용자의 질문만 보고 답변을 생성합니다.

 - 평가자 모드 (스포일러 당한 나): 에러 로그나 사용자의 핀잔을 미리 보고 "아, 이 라이브러리를 썼어야지!"라는 구체적인 힌트를 뽑아냅니다. 그리고 과거의 자신에게 돌아가 "이 단어의 확률은 높이고 저 단어는 낮춰라"라며 토큰 단위로 직접 교정합니다. 

쓸만한 힌트가 안 나오면 가차 없이 데이터를 버리는 깐깐함도 갖췄습니다.

3. 낮에 일하고 밤에 공부? NO! (비동기 파이프라인)

데이터를 며칠씩 모아뒀다가 밤에 몰아서 학습(Batch)하는 구조가 아닙니다.

사용자에게 답변을 서비스하는 동안, 뒷단에서는 방금 전 대답을 채점하고, 동시에 훈련 엔진은 가중치를 업데이트합니다. 

4개의 루프가 서로를 기다리지 않고 비동기로 쉼 없이 돌아갑니다. 

놀랍게도 단 24~36번의 상호작용만 거쳐도 에이전트의 말투나 습관이 사용자 입맛에 맞게 완전히 튜닝됩니다.

4. 감당할 수 없는 VRAM의 무게

이론은 너무나 아름답지만, 로컬 환경에서 실용적인 AI를 구축하려는 입장에서는 뼈아픈 현실이 숨어 있습니다.

서비스(Inference), 평가(PRM), 그리고 훈련(기울기 및 옵티마이저 상태 저장)을 동시에 돌리려면 VRAM이 말 그대로 폭발합니다. 

논문에서는 32B 모델을 64개 병렬로 돌리는 등 엄청난 클라우드 인프라를 사용했습니다. 

CPU 환경에서는 구동조차 불가능에 가깝습니다. 실시간으로 똑똑해지는 마법의 대가는 어마어마한 하드웨어 청구서인 셈입니다.

💡 마치며: 어떤 선택을 해야 할까?

먼저 OpenClaw가 만든 변화가 참 재밌습니다. 곧 OpenClaw is all you need라는 말이 유행할지도 모르겠습니다.

에이전트가 상호작용을 통해 스스로 성장하는 아키텍처는 분명 매력적이고 올바른 방향입니다. 

하지만 자원이 제한적인 로컬 환경이라면 이야기가 다릅니다.

차라리 낮에 사용자와 상호작용을 하며 데이터를 모아뒀다가, 밤에 데이터를 몰아서 학습하는 방식이 대부분의 사람들에게 더 나은 현실적인 구조이지 않을까 싶습니다.

모든 것은 거대한 GPU 클러스터가 있으면 더 좋지만, 없어도 많은 것들을 할 수는 있습니다. 다음에는 모델을 학습하지 않아도, 학습한 결과를 내는 논문을 소개드리겠습니다.

그리고 이 논문도 다음주 화요일이나 금요일에 진행할 Introduction to Post-Training 강연에서 짧게 다루겠습니다.
