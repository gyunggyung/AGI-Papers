그걸 설명하는 웹 페이지를 만들고, 직접 실행을 해보며 검증 까지 합니다. 


심지어 코딩을 하다 문법이 틀린 코드를 짜면 스스로 고치고, git commit까지 혼자서 합니다.

지금까지 제가 만든 Gemini-Claw가 수행한 일입니다. 

Gemini CLI 기반이라서 무료입니다. 앤트로픽에 수십, 수백, 수천만 원을 안 줘도 됩니다. (Free tier: 60 requests/min and 1,000 requests/day with personal Google account)

Gemini CLI 때문 같은데, 제가 구현한 기능 외의 일들을 수행하고 있어서 신기합니다. 

아직 구현하지 않은 기능들을 어떻게 스스로 하는지 조금 더 확인해보고 전부 이해한 후에, 공개를 할지 고민해보겠습니다. 참고로 아직 전체 코드가 2천 줄도 안 되는 것 같습니다.

다음으로 엑셀, 파워포인트, 워드를 만들고 조작하는 기능을 추가해볼까 합니다. 


노트북을 안 끄고 계속 돌리면 무언가 재밌는 결과가 나오지 않을까 싶습니다.

클로드 코드나 OpenClaw와 유사한 기능들을 구현하면 대단한 결과고, 아무것도 못 하고 끝날 가능성이 더 높기는 합니다. 

그리고 API 모델은 우리가 통제할 수 없습니다. 이걸 베이스라인 삼아서 오직 CPU 환경 온디바이스 LLM으로만 돌아가는 Tiny MoA를 발전시키려고 합니다. 

다만 시간과 돈 그리고 인력이 필요할 것 같습니다. 그래서 얼마나 할 수 있을지는 모르겠습니다. 


LLM은 유용한 도구입니다. 하지만 독점 기업들의 API 모델에 종속될 수는 없습니다. 그 통제권은 우리에게 있어야 합니다.

영상에 쓴 명령어는 다음과 같습니다.

uv run python -m src.main --query "적절한 이름의 새폴더 만들고 2601.15153v1.pdf 파일 내용의 핵심 기법을 python 파일로 구현하고 그걸 설명하는 웹 사이트를 html로 만들어"
