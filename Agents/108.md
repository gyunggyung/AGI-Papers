# Anthropic이 정의한 AI 에이전트 워크플로우 5선

> **"에이전트는 만능이 아니다. 상황에 맞는 '설계 패턴'이 필요하다." 앤트로픽이 제안하는 5가지 핵심 패턴.**

## 1. Prompt Chaining
작업을 순서대로 실행합니다. 문서 교정이나 번역처럼 단계가 명확할 때 좋습니다.

## 2. Routing
입력을 보고 가장 적절한 전문가 모델에게 토스합니다. 고객센터 챗봇 분류에 필수적입니다.

## 3. Parallel (병렬 처리)
여러 모델이 동시에 달려듭니다.
*   **Sectioning**: 긴 문서를 쪼개서 동시에 요약.
*   **Voting**: 여러 모델이 각자 코드를 짜고 투표로 결정(보안 점검 등).

## 4. Orchestrator-Workers
중앙 관리자(오케스트레이터)가 작업을 쪼개서 일꾼(워커)들에게 뿌리고, 결과를 취합합니다. 복잡한 프로젝트 코딩이나 리포트 작성에 적합합니다.

## 5. Evaluator-Optimizer
생성(Optimizer)하고 평가(Evaluator)하는 과정을 반복합니다. 루프를 돌면서 퀄리티를 극한으로 끌어올릴 때 사용합니다.

## 결론
무작정 복잡한 에이전트를 만들지 마세요. 내 문제가 '체인'으로 풀릴지, '병렬'로 풀릴지 먼저 파악하는 것이 엔지니어링의 시작입니다.
