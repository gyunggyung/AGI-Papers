랄프(Ralph) 같은 걸 따로 구현 안 해도 됩니다. Gemini나 클로드에게 개발 명세서와 테스트 케이스를 명확히 주고, "테스트 성공할 때 까지 코드 작성, 테스트 후 문제 파악하고 코드 개선, 다시 테스트 후 문제 파악 루프를 계속 돌아. 무한루프 진행." 이라고만 입력해도 성공할 때까지 계속 돌아가서 결국 해냅니다.

Gemini나 클로드가 코딩 하는 내용을 보다 보면, 가끔 이상한 방향으로 일을 하는 경우가 있습니다. 그럴 때 방향성을 한 번만 잡아주고 다시 무한루프를 돌리라고 하면, 다시 방향을 잘 잡아서 개발을 합니다. 물론 보기 귀찮으면 그냥 돌려놓고 다른 일을 해도 됩니다. 

요즘 정말 많은 플러그인들이 나오고 있지만, 개인적으로 대부분의 플러그인은 곧 사라질 것 같습니다.

더 중요한 것은 이미 똑똑한 LLM을 어떻게 더 효율적으로 쓸까 입니다. 이렇게 돌려도 클로드는 금방 죽고, Gemini도 금방 죽기에...

저는 노트북 CPU, 핸드폰, 라즈베리파이 그 어디 안에서든 동작하는 Liquid AI LFM2.5-1.2B-Instruct로 해당 기능을 구현해보려고 하고 있습니다.

그런데 생각보다 쉽지는 않고 슬슬 혼자 하는 작업에 한계를 느끼고 있습니다.

아무튼 추후 제 프로젝트에도 통합시켜보겠습니다!
