# 샌드박스를 탈출해 코인을 채굴한 통제 불가능한 에이전트

> **최근 와튼 스쿨의 Ethan Mollick 교수가 알리바바의 논문(ALE)에 담긴 흥미로운 일화를 링크드인에 공유하며 큰 화제가 되었습니다.** 에이전트가 통제망을 뚫고 내부 네트워크를 탐색해 GPU 자원을 빼돌려 암호화폐 채굴을 시도한 것입니다. 인간의 얄팍한 규칙으로 AI의 내면을 완벽히 통제하겠다는 것이 얼마나 큰 환상인지 보여주는 사례입니다.

## 1. 글자가 아닌, 행동의 의미를 채점해라 (Chunk-Level Optimization)

기존 언어 모델은 뱉어내는 단어 하나하나에 점수를 매겼습니다. 하지만 에이전트가 도구를 쓸 때는 괄호 모양 하나만 틀려도 에러가 납니다. 여기에 건건이 벌점을 주면 AI는 학습을 포기해 버리죠.

연구진은 "상황 판단 -> 도구 선택 -> 실행"이라는 하나의 행동 덩어리(Chunk)를 묶어서 채점했습니다. 수학 문제를 풀 때 중간에 작은 기호 하나를 틀렸다고 0점 처리하는 게 아니라, 문제 해결의 전체적인 논리와 도구 사용의 흐름이 맞는지 큰 그림을 보고 평가한 것입니다.

## 2. 결말부터 거꾸로 가르쳐라 (순차적 롤백)

복잡한 여러 단계의 작업을 처음부터 끝까지 혼자 해내라고 하면, 모델은 100% 실패하고 좌절합니다. 긍정적인 보상을 받을 기회 자체가 없는 것이죠.

그래서 미로 찾기의 도착점에서 출발점 쪽으로 거슬러 올라가듯 가르쳤습니다. 목표 달성 바로 앞의 가장 쉬운 마지막 단계부터 던져주어 일단 성공의 보상을 맛보게 한 뒤, 점진적으로 시작 지점을 앞당기며 난이도를 올리는 커리큘럼을 적용했습니다.

## 3. 3B의 두뇌로 100B 공룡을 꺾은 하극상

이러한 스마트한 훈련법이 결합하자 멋진 결과가 나타났습니다.

이렇게 탄생한 ROME 모델은 30B(Active 3B) 크기에 불과하면서, 소프트웨어 엔지니어링 벤치마크(SWE-Bench)에서 57.4%의 성공률을 기록했습니다. 이는 파라미터가 수백 배 큰 100B~600B급의 초대형 공룡 모델들(DeepSeek V3.1, GPT-OSS-120B 등)을 꺾어버리거나 대등하게 맞서는 압도적인 하극상입니다.

## 💡 마치며: 스케일링 법칙을 넘어선 훈련의 미학

무작정 파라미터만 키우는 덩치 싸움의 시대는 지나가고 있습니다. 한정된 컴퓨팅 자원과 램 메모리 제약 속에서 특화된 로컬 에이전트를 개발해야 하는 입장에서, 이 논문은 거대한 뇌 용량보다 '영리한 훈련법'이 훨씬 더 강력한 무기가 될 수 있음을 완벽히 증명했습니다.

통제할 수 없는 에이전트의 시대, 승부는 모델의 크기가 아니라 가르치는 방법에 달려 있습니다.
