---
GPT-5.5(9.7T), Claude Opus 4.6(5.3T) 등 다양한 독점 모델들의 파라미터를 유추하는 논문이 나왔습니다. 모델들의 유추 크기를 첨부했으니 이것만 보셔도 됩니다. 다만 그저 유추값입니다. 무엇을 근거로 이런 결과를 냈는지 설명해 드리겠습니다.

1. 핵심 아이디어: 지식은 압축되지 않는다

딥러닝 모델의 코딩이나 논리 추론 능력은 학습 기법이나 아키텍처 발전을 통해 적은 파라미터로도 효율을 높일 수 있습니다. 

하지만 "A라는 마이너한 대학교의 설립 연도는 B다" 같은 순수한 사실적 지식은 압축하거나 추론으로 때려 맞출 수 없습니다. 

즉, 모델이 이런 희귀한 사실을 많이 기억하고 있다면 그만큼 그것을 저장할 물리적인 파라미터(메모리) 용량이 크다는 것을 의미합니다.

2. IKP 벤치마크 설계

이를 측정하기 위해 논문 저자는 IKP(Incompressible Knowledge Probes)라는 1,400개의 질문 세트를 만들었습니다. 

인터넷에 널린 뻔한 상식이 아니라, 매우 희귀한 논문 저자의 세부 전공이나 잘 알려지지 않은 장소 등 7단계 난이도의 극단적인 롱테일(Long-tail) 지식들로만 구성해 모델의 진짜 정보 저장량을 찌른 것입니다.

3. 오픈소스 모델을 통한 기준점(공식) 도출

파라미터 크기가 이미 명확하게 공개되어 있는 135M부터 1.6T까지의 오픈소스 모델 89개에 이 테스트를 진행했습니다. 

테스트 결과, 희귀 지식에 대한 정답률과 파라미터 크기가 로그-선형(log-linear) 관계로 아주 일정하게 비례한다는 뚜렷한 공식을 발견했습니다.

4. 블랙박스 모델 크기 역산

이 공식을 이용해, 파라미터 수를 꽁꽁 숨기고 있는 API 기반 독점 모델(GPT-5.5, Claude Opus 4.6 등)의 크기를 추정했습니다. 

똑같은 1,400개의 희귀 질문을 던져 정답률을 알아낸 뒤, 앞서 만든 공식에 대입해 이 정도 정답률이 나오려면 파라미터가 대체 몇 개여야 하는지를 역으로 계산한 것입니다.

5. 왜 절대적이지 않은가?

모델이 정답을 알고 있어도 자체적인 안전 정책(RLHF 등) 때문에 답변을 거부하면 점수가 깎여 크기가 실제보다 작게 측정될 수 있습니다. 

또한, 특정 모델이 학습 데이터로 해당 지식을 우연히 더 많이 봤을 수도 있기 때문에, 이 수치는 정확한 파라미터 개수라기보다는 모델이 품고 있는 최소한의 유효 지식 용량(하한선)으로 이해해야 합니다.
