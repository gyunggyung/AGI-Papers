# LoRASA: 따로 또 같이, 에이전트의 개인기

> **"팀플레이(공유 정책)와 개인기(개별 적응)를 동시에 잡았다. LoRA로 만드는 가성비 멀티 에이전트."**

## 1. 딜레마

멀티 에이전트 강화학습(MARL)에서 모든 에이전트가 같은 정책을 쓰면(PS) 효율적이지만 개성이 없고, 따로 쓰면(NPS) 비용이 폭발합니다.

## 2. LoRASA의 해법

**LoRASA (Low-Rank Agent-Specific Adaptation)**는 거대한 공통 정책 모델 위에, 각 에이전트별로 아주 작은 **LoRA 어댑터**를 붙이는 방식입니다.

*   **공유**: 기본적인 행동 방침은 공유 모델이 담당합니다.
*   **특화**: 각 에이전트의 고유한 역할이나 스타일은 LoRA가 담당합니다.

## 3. 결과

*   **효율**: NPS 대비 파라미터를 **75%**나 아꼈습니다.
*   **성능**: StarCraft II 게임 등에서 기존 방식보다 월등한 성능과 적응력을 보여주었습니다.

## 4. 결론

"공유할 건 공유하고, 특화할 건 특화하라." 시스템 효율과 개별 에이전트의 전문성을 동시에 잡은 영리한 전략입니다.
