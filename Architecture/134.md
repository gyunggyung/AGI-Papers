title: "현대 LLM은 비쌉니다... Log-Linear Attention"
---

현대 LLM은 비쌉니다. 2K, 4K짜리 문서 수십 개만 동시 서빙하려고 해도 Gemma나 Qwen 같은 기존 트랜스포머 모델의 KV 캐시 비용은 언제나 살벌하게 비쌉니다. 이 메모리 비용을 아예 0으로 만들겠다며 등장한 대안이 Mamba나 Linear Attention 같은 순환 모델입니다. 

이들은 문장이 아무리 길어져도 과거의 모든 기억을 딱 한 장의 작은 칠판, 즉 헤드당 256행 256열 크기(1B 기준)의 고정된 행렬 하나에 계속 압축하고 누적하며 전진합니다. 길이가 늘어나도 메모리가 고정되니 이론상 무한 컨텍스트가 가능해 보였죠.

하지만 현실은 다릅니다. 실제 긴 문서나 복잡한 검색 테스트를 돌려보면, 이들은 기존 트랜스포머보다 훨씬 더 금방 기억력을 상실하고 무너져 버립니다. 대체 왜 그럴까요?

그리고 이를 구조적으로 해결하겠다는 Mamba와 FlashAttention 저자들의 연구 Log-Linear Attention을 소개합니다.

1. 칠판 한 장의 비극: 입력 용량 초과 (Capacity Failure)

이론상 무한히 읽을 수 있다던 모델들이 현실에서 무너지는 이유는 단순합니다. 압축을 받아낼 메모장(칠판)의 물리적 크기가 너무 작기 때문입니다.

수학적으로 256x256 크기의 행렬 하나에는 서로 100% 구별되는 핵심 정보(Key-Value 관계)를 최대 256개까지만 깨끗하게 담을 수 있습니다.

여기에 8K, 16K가 넘는 문서가 들어와 수백, 수천 개의 단어 매칭이나 비밀번호 쌍(MQAR)을 동시에 기억해야 한다면 어떨까요? 제한된 공간에 정보를 억지로 구겨 넣다 보니 숫자들이 서로 겹치고 뭉개지는 수학적 포화 상태가 발생합니다. 

답변을 쓰기도 전에 이미 입력 단계에서 머릿속이 엉망진창으로 포화되어 기억상실증에 걸리는 것입니다.

2. 가까이는 촘촘하게, 멀리는 대충: Log-Linear Attention

이 태생적 용량 제한을 극복하기 위해 "창고 하나로 부족하면, 창고의 개수 자체를 늘리자"는 아이디어에서 출발한 것이 Log-Linear Attention입니다.

다만 무작정 늘리면 예전 트랜스포머처럼 메모리가 터지겠죠? 그래서 문서 길이가 늘어나는 속도의 로그(Log) 스케일만큼만 칠판을 영리하게 늘려나가는 이진트리 구조를 채택합니다. 쉽게 말해 시간의 거리에 따라 기억의 해상도를 다르게 가져가는 전략입니다.

예를 들어 64K(약 6만 5천 토큰)의 긴 문서를 읽을 때, 내부에 총 16개(log2 65536 = 16)의 독립된 칠판을 장착합니다.

 - 1번 칠판: 방금 읽은 아주 최근의 단어들을 토큰 단위로 아주 촘촘하고 정확하게 적습니다.

 - 중간 칠판들: 시간이 꽤 지난 단어들을 4개, 8개씩 묶어서 대강 압축해 적습니다.

 - 16번 칠판: 아주 먼 옛날에 읽은 첫 페이지 내용들을 거대한 블록으로 뭉뚱그려 적습니다.

공간을 계층적으로 분할하여 정보를 분산시키니, 엄청나게 많은 정보가 밀려 들어와도 뭉개지지 않고 끝까지 살아남습니다. 기존 순환 모델의 치명적인 용량 한계를 아키텍처적으로 치료한 셈입니다.

3. 수학적 우아함 뒤에 숨은 냉혹한 하드웨어 청구서

하지만 세상에 공짜는 없습니다. 로그 스케일이라는 화려한 포장지 뒤에는 아주 살벌한 기회비용이 숨어 있습니다.

 - 초고속 캐시(SRAM) 이탈과 속도 저하: 현대 AI 칩이 미친 듯이 빠른 이유는 칩 내부의 아주 작고 빠른 캐시 메모리(SRAM) 안에 칠판 1장을 딱 고정해두고 연산하기 때문입니다. 하지만 칠판이 16개로 늘어나면 이 캐시 용량을 초과해 버립니다. 

결국 연산할 때마다 메인 메모리(VRAM)로 데이터를 넣었다 뺐다 하는 극심한 교통정체(I/O 병목)가 발생해 텍스트 생성 속도가 문맥이 길어질수록 뚝뚝 떨어집니다.

 - 배치 사이즈(Batch Size)의 저주: 서비스 환경에서는 사용자 1명이 아니라 수백 명의 요청을 동시에 처리해야 합니다. 동시 처리 인원이 128명이고 64K를 읽는다면, 동시에 유지해야 하는 칠판이 16장 x 128명 = 2,048장으로 폭발합니다. 값비싼 서버 메모리가 순식간에 바닥납니다.

 - 기존 자산의 전면 폐기: 구조가 바뀌는 순간, 기존 오픈소스 생태계가 엄청난 돈을 들여 학습시켜 놓은 기존 모델들의 훌륭한 가중치들을 재사용할 수 없습니다. 거대한 Pre-training 비용을 고스란히 감당해야 합니다.


마치며

이 논문은 그동안 순환 모델 진영이 애써 외면해왔던 고정된 저장 공간의 한계(Fixed-State Capacity Bottleneck)를 정면으로 돌파하고 실질적인 해결책을 제시했습니다.

저자들은 자신들의 모델이 트랜스포머를 모든 면에서 압도했다고 사기 치지 않았습니다. "기억 용량을 얻은 대신, 하드웨어 추론 비용이 늘어난다"는 한계를 수학적, 실험적으로 투명하게 인정했습니다. 

Log-Linear Attention은 순환 모델이 가진 고질적인 기억 용량 부족 문제를 시간축 트리 구조라는 수학적 미학으로 풀어낸 잘 쓴 논문입니다. 한 번 봐보시는 것을 추천드립니다.
