"1.2B 모델이 유용한 PPT를 만들 수 있습니다."

Tiny-MoA에 Office(Word, Excel, PPT) 생성 기능을 추가했습니다. API 비용 0원으로, 내 노트북에서 돌아가는 1.2B 모델(LFM2.5)로 이걸 구현했습니다. 

사실상 해당 작업은 Liquid AI LFM2.5 1.2B 모델 중심입니다. (따로 빼서 이것만 잘하게 만드는 에이전트를 만들까도 고민 중입니다.)

Gemini나 Claude로 고품질 제안서 만드는 건 쉽습니다. API만 쏘면 되니까요. 하지만 인터넷도 안 되는 1.2B 로컬 모델로 컨설팅 펌 수준의 보고서를 만들 수 있을까요?

Tiny-MoA의 오피스 자동화(PPT, Word, Excel 생성) 기능을 고도화했습니다. 목표는 단순했습니다.


제가 만든 Gemini-Claw를 벤치마킹해서, 1.2B 모델인 Liquid AI LFM2.5-1.2B-Instruct에 이식해봤습니다.

Gemini-CLI를 쓰는 Gemini-Claw에 범용 오피스 생성 기능을 추가하는 데 고작 30분이 걸렸습니다. 그런데 로컬 LLM은 이미 구조를 아는 상태였음에도, 불안정하게나마 구현하는 데 2시간이나 걸렸습니다.

로컬 소형 모델(sLLM) 다루기는 확실히 어렵습니다. API 딸깍하는 순간 그리웠습니다. 물론 제가 새로운 아키텍쳐를 만들어서 Pre-training부터 했을 때는, Liquid AI 모델을 쓰는 순간이 그리웠습니다. 모든 것은 상대적인 것이죠.

🎯 도전 과제
1.2B 모델도 똑똑하지만 JSON 형식을 자꾸 틀립니다. "PPT 만들어줘" 하면 텍스트만 주절주절 뱉습니다. 전문적인 용어? 기대하기 힘들었죠.

🛠️ 해결책: "프롬프트 깎는 노인"이 되다
작은 모델일수록 '개떡같이 말하면 개떡같이 알아듣습니다'. 찰떡같이 말해야 합니다. LFM2.5 모델은 이미 28T token을 Pre-training한 모델입니다. 거기다 지시이행(IFBench) 점수는 Claude-3.5 Sonnet 이상입니다. 이미 지식이 있으니, 그 능력을 극대화 시켜야 합니다.

1. 페르소나 주입 (Persona Injection)
 - "너는 AI야" 대신 "너는 Top-tier 전략 컨설턴트야"라고 최면을 걸었습니다.
 - 효과는 즉각적이었습니다. 말투가 "도와드릴까요?"에서 "본 제안서는~"으로 바뀌고, 구조적 사고를 하기 시작합니다.

2. 컨텍스트 강제 주입 (Context RAG)
 - 모델이 환각을 일으키지 않도록, 'README.md' 같은 로컬 프로젝트 파일을 강제로 읽혀서 주입했습니다.
 - 추후에는 docling을 이용해 핵심 파일(PDF 등)을 자동 탐색하는 기능도 추가할 예정입니다. (이미 PDF 읽기는 됩니다)

3. 구조적 프롬프트 (Structured Prompt)
 - [TASK], [CONTEXT], [FORMAT], [EXAMPLE] 섹션을 나누어 숟가락으로 떠먹여주듯 프롬프트를 짰습니다.
 - 특히 JSON 예시(Example)를 보여주니 그제야 말을 듣더군요. (LFM2.5 모델 특성에 맞춰 포맷을 강제했습니다.)

📊 결과 (The Output)
이제 Tiny-MoA는 스스로:
- 📑 PPT: 5장 내외의 구조화된 슬라이드 (제목+개조식 본문)
- 📝 Word: 요약, 문제 정의, 해결 방안이 담긴 상세 보고서 (글머리 기호 자동 적용)
- 📊 Excel: 그럴듯한 Mock Data와 스타일링이 적용된 스프레드시트
를 생성합니다. 물론 100% 로컬에서요.

📉 한계와 교훈 (Reality Check)
솔직히 한계는 명확합니다. 현재는 '상용 API'의 퀄리티를 흉내 내기 위해 '자기소개(Tiny-MoA 프로젝트)'에 과적합(Overfitting)되어 있습니다. 범용적인 주제(예: 여행 계획)를 시키면 아직 헤맵니다. (곧 확장할 예정입니다.)

하지만 결론은 명확합니다. "상용 API로 구현 가능한 기능은, 로컬 LLM으로도 (노력을 갈아 넣으면) 가능은 하다."

여러분이 클로드 코드를 쓰면서, 가장 안전하다고 떠드는 클로드가 우리의 명령을 무시하는 경우를 못 보셨나요? 애초에 독점 기업들의 모델들은 완벽하게 안전하지 않습니다.

우리가 통제할 수 있는, 튜닝할 수 있는, 제어할 수 있는, 온디바이스(로컬) LLM이 필요합니다.

거대 테크 기업들이 API 가격을 올리거나 기술을 비공개로 전환한다면, 우리에겐 우리가 통제할 수 있는 모델이 있어야 합니다. 제가 생각하는 미래의 AI 시스템은 하나의 거대한 뇌가 아니라, 작지만 전문적인 여러 모델이 톱니바퀴처럼 맞물려 돌아가는 형태입니다. 제 Tiny MoA처럼 말이죠.
